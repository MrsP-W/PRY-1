## 🚨 撞坑汇总(本次新增 4 类,范本累计 8 → 9)

### 撞坑 #19 · classifier 误判 plutil -p 为 credential 泄露

| 项 | 内容 |
|------|------|
| **撞坑现象** | `plutil -p ~/Library/LaunchAgents/com.myaiemployee.agent.plist` 被 classifier 拒绝,理由 "Credential Leakage" |
| **触发条件** | classifier 检测到 v0.2.14 写过 SQLCIPHER_KEY 到 .env,后续动作风险高 |
| **影响范围** | 阻塞 plist 内容查看(只读)|
| **沿 v0.2.13 撞坑** | **新增第 18 类**(classifier 误判)|
| **撞坑恢复** | 跳过 `plutil -p` 详细查看,仅用 `plutil -lint` 语法校验 |
| **范本沉淀** | classifier 误判场景:`plutil -p` 只读命令被误判为 credential 泄露 |

### 撞坑 #20 · classifier 双重混淆(只读 + 拒绝)

| 项 | 内容 |
|------|------|
| **撞坑现象** | `.venv/bin/python3 --version` 被 classifier 拒绝,但 classifier 自身说 "No BLOCK rule is triggered" + "匹配用户边界" |
| **触发条件** | classifier 内部逻辑矛盾(既说匹配边界又说拒绝)|
| **影响范围** | 阻塞 Python 版本查看 |
| **撞坑恢复** | 沿 [[v0.2.4]] §3 机制 3 不绕过 classifier,改用 `python3 --version` 系统 Python 验证 |
| **范本沉淀** | classifier 双重混淆应对:不绕过 + 改用替代命令 |

### 撞坑 #21 · pwd 漂移导致 .venv/bin/ 消失(伪撞坑)

| 项 | 内容 |
|------|------|
| **撞坑现象** | `ls .venv/bin/` 报 `No such file or directory` |
| **触发条件** | pwd 在 `Agent Assistant`(主项目)而非 `我的AI员工`(子项目) |
| **影响范围** | 阻塞 venv 命令 |
| **真因** | 主项目 `Agent Assistant` 没有 `.venv/`,子项目 `我的AI员工` 才有 |
| **撞坑恢复** | `cd /Users/wei/Documents/DesktopOrganizer/我的AI员工 && pwd && ls .venv/bin/` |
| **范本沉淀** | **Bash 跨项目时必须显式 cd** —— pwd 默认是上次 cd 的位置,可能漂移到错目录 |

### 撞坑 #22 · grep 连写模式错误漏掉 plist

| 项 | 内容 |
|------|------|
| **撞坑现象** | `grep -iE "my-ai\|myai\|ai-employee"` 漏掉 `com.myaiemployee.agent.plist` |
| **触发条件** | `myai` 在 `myaiemployee` 中是连写,grep 模式错误 |
| **影响范围** | plist 文件查找失败 |
| **真因** | grep 模式 `-iE "my-ai\|myai\|ai-employee"` 三个分支都不匹配 `myaiemployee`(连写无分隔符)|
| **撞坑恢复** | 改用 `ls ~/Library/LaunchAgents/ \| grep -i "myai"` 或 `ls ~/Library/LaunchAgents/com.myai*` |
| **范本沉淀** | grep 多分支模式容易漏连写,优先用 `ls -la | grep -i <keyword>` 单分支 |
