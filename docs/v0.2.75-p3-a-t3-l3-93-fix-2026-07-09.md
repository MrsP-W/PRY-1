## 🎯 撞坑 #93 — launchd 子进程 PATH 不含 uv

### 现象

T3 L3 `launchctl load -w com.myaiemployee.digital-employee.plist` 后:

```text
launchctl list | grep myaiemployee

~/Library/Logs/MyAIEmployee/digital-employee.out.log:
```

### 关键诊断

撞坑 #92 修复 B **完全生效**:`.env` 读 APP_SUPPORT 路径成功,`DB_ENCRYPTION_KEY` + `Keychain` 全部加载。

但 `preflight_check` 的 `uv run alembic current` 和 `uv run python -c "import dashboard.server"` 因为找不到 `uv` 报 warning(本应 OK)。然后 `cmd_start_menubar` 的 `nohup ... uv run python ...` 同样找不到 `uv` → nohup 失败 → 菜单栏 PID 文件被清理 → wrapper exit 1。

### 根因

```text
launchd 默认 PATH:/Users/wei/bin:/usr/local/bin:/usr/bin:/bin
uv 安装位置:        /opt/homebrew/bin/uv(Homebrew Apple Silicon 标准位置)

→ launchd 子进程 PATH 不含 /opt/homehomebrew/bin,uv 找不到
```

macOS launchd 设计上**不继承**用户 shell `~/.zshrc` / `~/.zprofile` 的 PATH 扩展。任何"用户 shell 自定义"(brew shellenv / pyenv / asdf / 自定义 PATH)都不能直接进入 launchd 子进程。

### 与撞坑 #91 / #92 区分

| 维度 | #91 | #92 | #93 |
|------|-----|-----|-----|
| 触发位置 | launchd exec 阶段 | runtime 业务代码 | launchd 子进程 PATH |
| 拦截源 | macOS Documents 沙箱 | macOS Documents 沙箱 | launchd minimal PATH |
| 修复路径 | wrapper 改 `~/bin/` | runtime 路径迁 APP_SUPPORT | 绝对路径 + UV_BIN 检测 |

三者共同特点:**user shell 配置不能直接迁移到 launchd 子进程**。每个撞坑都是 macOS launchd 生态本身的限制暴露。

### 2. `ops/start-digital-employee.sh` digital-runner — `${UV_BIN}` 检测(方案 C)

脚本顶部(紧跟 `# 启动入口` 段后):

```bash
# 撞坑 #93 修复(2026-07-09):launchd 子进程 PATH 不含 /opt/homebrew/bin(uv 安装位置),
# 用 command -v 优先探测 PATH · fallback 绝对路径,沿 v1.0 launch runbook 范本(可移植)
UV_BIN="$(command -v uv 2>/dev/null || echo /opt/homebrew/bin/uv)"
```

替换 6 处 `uv run` 调用:

| 位置 | 旧 | 新 |
|------|----|----|
| precheck alembic (line 195) | `uv run alembic current` | `"${UV_BIN}" run alembic current` |
| precheck dashboard import (line 211) | `uv run python -c "import..."` | `"${UV_BIN}" run python -c "import..."` |
| real nohup menu bar (line 264) | `nohup env ... uv run python "$RUN_MENUBAR"` | `nohup env ... "${UV_BIN}" run python "$RUN_MENUBAR"` |
| real nohup dashboard (line 297) | `nohup env ... uv run python -m my_ai_employee.dashboard.server` | `nohup env ... "${UV_BIN}" run python -m my_ai_employee.dashboard.server` |
| dry-run echo menu bar (line 258) | `echo "[dry-run] ... uv run python ..."` | `echo "[dry-run] ... ${UV_BIN} run python ..."` |
| dry-run echo dashboard (line 291) | `echo "[dry-run] ... uv run python ..."` | `echo "[dry-run] ... ${UV_BIN} run python ..."` |

### 3. `tests/scripts/test_launchd_install.py` — 新增 4 测试

- **H1**:`ops/start-digital-employee.sh` 必含 `UV_BIN="$(command -v uv 2>/dev/null || echo /opt/homehomebrew/bin/uv)"`
- **H2**:所有非注释行的 `uv run` 调用必用 `${UV_BIN}`(至少 6 处:2 precheck + 2 real nohup + 2 dry-run echo)
- **H3**:monthly-report wrapper heredoc 必含 `/opt/homebrew/bin/uv run --project`
- **H4**:imap-sync wrapper heredoc 必含 `exec /opt/homebrew/bin/uv run --project`

测试结果:`47 passed in 0.04s`(原 43 + 新 4 = 47)。

H2 测试细节:用 regex `r"\$\{UV_BIN\}[\"']?\s+run\b"` 兼容两种形式(quoted `"${UV_BIN}" run` 与 unquoted `${UV_BIN} run`)。
