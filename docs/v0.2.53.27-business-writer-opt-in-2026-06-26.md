# v0.2.53.27 BusinessWriterImpl opt-in 注入(2026-06-26)

> **承接**:v0.2.53.18 `DashboardContext.with_business_writer()` 不可变更新公共 API + v0.2.53.17 `BusinessWriterImpl` 接入骨架
> **目标**:`BUSINESS_WRITER_ENABLED=1` + `DASHBOARD_REAL_DB=1` + session_factory OK → `DashboardContext.default()` 自动注入 Impl
> **沿用**:`DASHBOARD_REAL_DB=1` 范本(v0.2.53.7-8)+ 撞坑 #65 opt-in 4 阶段范本

## 1. 目标

把 v0.2.53.17 的 `BusinessWriterImpl` 接入骨架 + v0.2.53.18 的 `DashboardContext.with_business_writer()` 公共 API 串起来,
让 `DashboardContext.default()` 在用户显式 `BUSINESS_WRITER_ENABLED=1` 时自动注入 Impl,
而**默认仍走 Stub**(沿撞坑 #65 默认 Stub 边界)。

不动:ApprovalGate 决策矩阵、handler 路径 3.5、`write_executed` 恒 False 不变式、4 类动作方法默认 raise。

---

## 2. 实现路径

### 2.1 `context.py` 新增(沿 v0.2.53.7 范本)

**模块顶部新增常量**:
```python
# v0.2.53.27 opt-in BusinessWriterImpl env 门控(沿 DASHBOARD_REAL_DB=1 范本)
_BUSINESS_WRITER_ENABLED_ENV = "BUSINESS_WRITER_ENABLED"
```

**新增 helper `_is_business_writer_enabled()`**:
沿 `_is_real_db_enabled()` 严判范本,只识别 truthy 字面量 `{"1", "true", "yes", "on"}`。

**新增 helper `_try_build_business_writer_from_session_factory()`**:
单 `try / except Exception`,失败返回 `None`(沿 v0.2.53.8 单项失败降级)。

**`DashboardContext.default()` 末尾追加**:
```python
if _is_business_writer_enabled():
    writer = _try_build_business_writer_from_session_factory(session_factory)
    if writer is not None:
        ctx = ctx.with_business_writer(writer)
```
