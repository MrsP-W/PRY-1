## 3. 撞坑沉淀

### 3.1 撞坑 #57(本轮新增)— ast.unparse() 注释丢失陷阱

**现象**:Python `ast.unparse()` 用于"批量加泛型参数"会丢失注释 / `# type: ignore` / 引号风格 / 空行。

**范本**:**AST 转换** = 不安全的批量修改工具,应优先用**正则**(只替换精确模式)而非 `ast.unparse()` 整文件重写。

**教训**:**任何批量 AST 修改必须保留原始注释 + 类型 ignore + 引号风格** — 否则会引入大量回归。
