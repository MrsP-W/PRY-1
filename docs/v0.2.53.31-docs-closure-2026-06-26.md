## 2. 撞坑沉淀 · 撞坑 #50 第三层范本衍生

### 撞坑 #50 衍生(本轮新增撞坑史 7 类)· "实施后 docs 补档窗口"撞坑

**问题**:v0.2.53.30 commit `915c7f8` 落地 9 文件 + 三入口同步,但 **docs/ 目录缺 v0.2.53.30 8 段设计稿**,导致 docs 目录与 SESSION-STATE 顶部状态行不一致(v0.2.53.26/27/29 都有 docs 配套,v0.2.53.30 跳号)。

**根因**:
- v0.2.53.30 commit 优先级 = "feat + 三入口同步"为主,docs 8 段设计稿未进入 commit 链(沿 v0.2.51 docs-only 范本拆分,但本轮拆分不彻底)
- 撞坑 #50 第三层范本仅覆盖 baseline 同步(quality_snapshot + tests 硬编码),未覆盖 docs 目录配套

**解决**:
- 沿 v0.2.53.25 docs-only 范本,v0.2.53.31 docs 收口 1 docs-only commit 补档
- 撞坑 #50 范本升级:从"baseline 同步第三层"扩展为"docs 目录配套第三层"(任何 feat commit 在 1-3 docs-only commit 内必须补 docs/ 配套设计稿)

**范本升级**:撞坑 #50 状态漂移第三层范本衍生 = "实施后 docs 补档窗口"——commit 落地后,docs 8 段设计稿应在 1-3 docs-only commit 内补档。

**Why**:v0.2.53.27 + v0.2.53.29 都有 docs 8 段设计稿同步 commit,v0.2.53.30 首次出现"feat 9 文件 + 三入口同步但 docs 8 段未补"差异,导致新会话读 docs/ 时跳号。

**How to apply**:沿 v0.2.53.25 范本,任何 `feat(...): v0.2.XX.YY ...` commit 在 1-3 docs-only commit 内必须补 `docs/v0.2.XX.YY-...md`,不能仅靠 SESSION-STATE 顶部状态行作为"实施落地"凭证。
