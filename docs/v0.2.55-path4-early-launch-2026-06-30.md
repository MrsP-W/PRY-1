## 1. 本次实际落地

### 1.1 五门写入条件

Path 4 真实写入现在必须同时满足:

| 门 | 条件 | 失败结果 |
|----|------|----------|
| 1 | `DASHBOARD_WRITE_API=1` | `403 write_disabled` |
| 2 | POST `confirm_text="CONFIRM_WRITE"` | `403 confirmation_required` |
| 3 | `BUSINESS_WRITER_ENABLED=1` 且 `BusinessWriterImpl` 已注入 | `501 write_not_implemented` |
| 4 | `real_write_handler_enabled=True` | `501 write_not_implemented` |
| 5 | `ENABLE_PATH_4_WRITE=1` | `501 write_not_implemented` |

另外,默认 `DashboardContext.default()` 仍要求 `DASHBOARD_REAL_DB=1` 才会尝试构造真实 session/store;否则保持 Stub。

### 1.2 代码路径

| 文件 | 变更 |
|------|------|
| `src/my_ai_employee/dashboard/approval_gate.py` | `dry_run=false` 在前三门通过后返回 200,交给 handler 执行 Path 4 分发 |
| `src/my_ai_employee/dashboard/handlers.py` | 新增 `_execute_writer_action()` 与 `_call_writer_action()`;成功返回 `read_only=false` / `write_executed=true` / `affected_id` / `audit_id` |
| `src/my_ai_employee/dashboard/business_writer_impl.py` | 新增 `ENABLE_PATH_4_WRITE` 第 5 门;4 动作入口按"参数 -> 写保护锁 -> 依赖 -> service -> audit"顺序执行 |
| `src/my_ai_employee/dashboard/context.py` | 注入 `OutboxStore` / `NoteConfirmServiceImpl` / audit_store,并把第 5 门同步传入 writer |
| `src/my_ai_employee/dashboard/responses.py` | `/api/status` 暴露 `enable_path_4_write_env_enabled` 与 `path4_write_ready` |

### 1.3 仍然保持拒写的路径

- 默认无 env:不写。
- 只开 `ENABLE_PATH_4_WRITE=1`:不写,仍卡第 4 门。
- 只开前三门:handler 会进入 writer,writer 仍卡第 4/5 门。
- `finance.dismiss_anomaly`:当前没有真实 `AnomalyDismissalServiceImpl` 注入,五门全开后仍不会伪装成功。
- service 抛异常:handler 返回 `409 write_failed`,writer 负责记录失败 audit。
