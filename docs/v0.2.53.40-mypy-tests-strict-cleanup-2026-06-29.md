## 1. 撞坑识别

### 1.1 真实错误数 vs 预期错误数

| 维度 | 预期 | 实际 | 差距 |
|------|------|------|------|
| 总错误数 | 13 | **300** | **+23x** |
| 错误类型 | 1 类(unused-ignore) | **3 类** | 类型严重低估 |
| 错误分类 | 单一 | 278 unused-ignore + 14 no-any-return + 8 misc | 缺 type 严判 / cast / 子类继承 3 类别 |

**根因**:
- v0.2.40 启用 `disallow_untyped_defs = true` 时遗留 `type: ignore` 注释(用于压制 mypy 旧警告)
- v0.2.41 启用 `mypy --strict` 后,这些注释变为 unused(已无对应警告)
- 但 v0.2.41-42 没顺手清 `# type: ignore`,且新加的 no-any-return + misc 错误也未处置
- 累积到 v0.2.53.40 共 300 errors

**撞坑**:**撞坑 #69**(本轮新增)**type: ignore 注释漂移积累** — 一旦启用更严格的 mypy 配置,旧的 `# type: ignore` 注释会大规模失效(unused-ignore),且会暴露之前被掩盖的 no-any-return / misc 错误。

### 1.2 三类错误根因

| 错误类 | 数量 | 根因 | 修复范本 |
|--------|------|------|----------|
| **unused-ignore** | 278 | `# type: ignore[xxx]` 已无对应警告(mypy 严格化后自动消失) | 直接删除注释(脚本批量)+ 4 处 `# noqa` 邻接手动修 |
| **no-any-return** | 14 | 返回值是 `Any` 类型变量(OutboxStore / NoteStore 等 Repository 返回的 entity)| `cast(int, entry.id)` 范本(沿 v0.2.23 `cast` 范本)|
| **misc** | 8 | `class X(SomeAnyTypedClass)` — 父类被 mypy 推断为 `Any`(动态导入 / cast)| `class X(SomeAnyTypedClass):  # type: ignore[misc]` 范本 |

## 2. 设计要点

### 2.1 三层修复策略

```
Layer 1: 自动化(278 unused-ignore)
  └── /tmp/fix_unused_ignore_safe.py
      - 跑 mypy 拿 [unused-ignore] 错误列表
      - 单文件 readlines(keepends=True) 读取
      - regex 匹配 `\s*# type: ignore[xxx]\s*$`
      - 单行替换 + writelines 写回(逐个文件原子操作)
      - 4 处 `# noqa` 邻接漏修(后续手动)

Layer 2: 范本化(14 no-any-return)
  └── cast 范本(沿 v0.2.23)
      - 5 文件 imports 添加 cast
      - `return entry.id` → `return cast(int, entry.id)`
      - 1 文件 `store.is_blocked(email)` → `cast(bool, ...)`
      - 1 文件 `build_status_payload(...)` → `cast(dict[str, Any], ...)`

Layer 3: 类型守卫(8 misc)
  └── `# type: ignore[misc]` 范本
      - 5 父类被推断 Any 的子类(FakeIMAPConnector / FakeReviewResult 等)
      - 加在 class 定义行尾 + noqa 注释前
```

### 2.2 安全约束

- **不前进 quality_snapshot.py** — 沿撞坑 #50 第三层防御(质量门基线已 2518/88.78%/177/0)
- **不动 ApprovalGate / 业务 writer / DB / SMTP** — 沿 v0.2.53.15-27 边界
- **不前进 9 质量门** — 仅消除 mypy errors 不增测试

### 2.3 脚本设计要点(`/tmp/fix_unused_ignore_safe.py`)

- **`readlines(keepends=True)`** — 关键:每个 line 都保留 `\n`,避免 `writelines` 合并行
- **`TYPE_IGNORE_END_RE = re.compile(r"(\s*)#\s*type:\s*ignore(?:\[[^\]]+\])?\s*$")`** — 用 `$` 锚定行尾 + `re.search` 而非 `re.sub`(避免行中误伤)
- **逐错误处理 + 每次重读文件** — 处理多错误同文件场景
- **失败标记 `FAILED:`** — 4 处 `# noqa` 邻接自动跳过,后续手动处理
