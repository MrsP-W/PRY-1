## 5. ⚠️ 已知限制(Constraints in v0.1)

### 5.1 数据库加密

- **生产 DB**:SQLCipher 4 加密(`~/Library/Application Support/my-ai-employee/data.db`)
- **测试/开发**:`--no-encrypt` 模式走明文 sqlite(e2e + dev 用)
- **风险**:DB 密码忘记 → 数据不可恢复(无密码找回机制)

### 5.2 平台限制

- **仅 macOS**:AppleScript(Notes 同步)+ TCC 权限(剪贴板监听)+ Keychain(密码存储)
- **不支持** Linux / Windows(沿 CLAUDE.md macOS-only 决策)
- **macOS 26 Tahoe 兼容**:rumps NSStatusBar 兼容测试未跑(留 v0.2 验证)

### 5.3 LLM 依赖

- **minimax M3**(默认)+ 可切换其他 provider
- **真实 LLM 调用需 API Key**(`data/llm_audit.log` 记录)
- **降级路径**:LLM 不可用时降级到规则引擎(关键词分类 + 模板化草稿)

### 5.4 微信/支付宝导入

- **CSV 解析版本**:2024 / 2025 / 2026 三版本字段(识别率 100%)
- **未知版本**:`UnsupportedCSVVersionError` 退出 1(沿 D6.6 P1 修复)
- **跨源去重**:**绝不**自动 merge,只标记 `needs_confirm=True`(沿 D7 决策)

### 5.5 笔记隐私

- **私密笔记**:`is_private=True` → PrivateSkipDecisionReport(业务硬阻断,沿 D4.7.2 v1.0.6 范本)
- **不传 LLM**:本地直存 SQLite,跳过 structure_and_emit
- **TCC 权限**:首次运行需授权"系统设置 > 隐私与安全 > 自动化 > Notes"

### 5.6 launchd 部署

- **仅 macOS**:`launchd_install.sh` 在 Linux/Windows 不工作
- **macOS 13+**:`x-apple.systempreferences:` URL 协议工作
- **macOS Sequoia TCC**:`~/bin/` 部署必为完整文件复制(非软链,沿 D5.6 memory)

## 7. 🛡️ 风险缓解(沿 9 范本)

| # | 风险 | 等级 | 缓解 | 落地 |
|---|------|------|------|------|
| 1 | 4 重防误发(SMTP 误发) | 🚨 严重 | `--to-whitelist` + `--max-messages 1` + `--confirm` + `--count 1` + `SMTP_REAL_NETWORK=1` | D5.6.5 真实 1 封范本 |
| 2 | 跨源去重误合并 | 🚨 严重 | 3 层去重模型(同源 UNIQUE + 跨源 INDEX + needs_confirm 软标记),**绝不**自动 merge | D6+D7 决策 |
| 3 | LLM 草稿被提示注入 | ⚠️ 中 | `UNTRUSTED_DATA_BEGIN/END` 包裹 + `json.dumps(ensure_ascii=True)` + `_MAX_BODY_CHARS=2000` 抗截断 | D4.7.2 v1.0.6 |
| 4 | DB 锁导致业务失败 | ⚠️ 中 | `except OperationalError: raise`(透传 → exit 3,沿 D3.3.3 + D9.6.3 教训) | D9.6.3 sync_notes.py 异常收窄 |
| 5 | AppleScript TCC 权限 | ⚠️ 中 | 首次运行引导"系统设置 > 隐私与安全 > 自动化" + URL 协议 `x-apple.systempreferences:...` | D9.5 tcc.py |
| 6 | launchd 误装载到 Documents/ | 🚨 严重 | 必部署到 `~/bin/`(macOS Sequoia TCC 拦截 Documents/),D5.6 范本 | D10.3 install.sh |
| 7 | LLM 调用 > 5000ms | ⚠️ 中 | 审计红线 + 降级到规则引擎 | 审计员 + S6.1/S6.2 范本 |
