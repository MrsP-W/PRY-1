## 1. 一句话总结

OutboxDispatcher 暴露公共 `active_provider` + `provider_defaults` 属性(沿 v0.2.52.2 ProviderDefaults 封装硬化范本),与 `EmailSendAdapter.provider_defaults` 双端对称封装;**5 处私有属性断言迁移到公共 API**(`test_outbox_dispatcher.py` 3 处 + `test_send_adapter.py` 2 处),测试侧不再读 `_active_provider` / `_provider_default_*` 私有字段。

## 2. 背景与动机

### 2.1 v0.2.52.2 留下的缺口

v0.2.52.2 `0955f2e` 已将 `EmailSendAdapter` 的 provider 默认配置字段封装为:
- `smtp_provider` 公共只读属性
- `provider_defaults` 公共只读属性(返回 `ProviderDefaults` dataclass)
- 内部仍保留 `_smtp_provider` / `_provider_default_host/port/email` 私有字段(供构造时赋值 + 运行时使用)

OutboxDispatcher 已**改读公共 API**(L337-343 in v0.2.52.2):
```python
adapter_provider = send_adapter.smtp_provider  # 公共属性
provider_defaults = send_adapter.provider_defaults  # 公共属性
```

但 **OutboxDispatcher 自己却没有暴露对应的公共 API**,测试侧仍读:
```python
dispatcher_local._active_provider
dispatcher_local._provider_default_host
dispatcher_local._provider_default_port
dispatcher_local._provider_default_email
```

这造成:
1. **封装不对称** — Adapter 有公共 API,Dispatcher 没有
2. **测试侧读私有字段** — 违反 encapsulation 范本
3. **重构风险** — 若未来 Dispatcher 内部字段重命名,5 处测试断言全部失效

### 2.2 v0.2.52.3 修复方向

**沿 v0.2.52.2 范本**:OutboxDispatcher 也暴露公共 API + 测试侧迁移到公共 API + 与 Adapter 双端对称。

---

## 3. 实现要点

### 3.1 OutboxDispatcher 暴露公共属性

`src/my_ai_employee/scheduler/outbox_dispatcher.py:402-419` 新增两段:

```python
@property
def active_provider(self) -> str | None:
    """当前激活的 SMTP provider 名(qq/outlook/gmail);未启用 provider 模式时为 None.

    v0.2.52.3 暴露公共 API:测试与外部观察者无需读私有字段 `_active_provider`。
    """
    return self._active_provider

@property
def provider_defaults(self) -> ProviderDefaults:
    """provider 默认 host/port/email 只读快照(供下游参考,避免读私有字段).

    v0.2.52.3 暴露公共 API:与 EmailSendAdapter.provider_defaults 对称封装。
    """
    return ProviderDefaults(
        host=self._provider_default_host,
        port=self._provider_default_port,
        email=self._provider_default_email,
    )
```

## 4. 撞坑 #64(本轮新增)— 测试侧公共 API 迁移范本

**范本**:任何"上游组件已暴露公共 API,下游组件内部封装但未暴露公共 API"场景:
① 上游暴露公共属性(沿 [[v0-2-52-2-smtp-provider-factory-integration]] 范本)② 下游对称暴露公共属性(沿 [[v0.2.52.1-outbox-dispatcher-auto-routing]] 撞坑 #63 范本)③ 测试断言统一迁到公共 API(避免读私有字段)④ 私有字段仍保留供内部赋值(encapsulation)⑤ 与上游 dataclass 类型对齐(`ProviderDefaults` 共享)⑥ 沿 v0.2.51 #18 风险门控 + v0.2.52 #61 协议修复 + v0.2.52.2 封装硬化范本三层叠加。

### 4.1 范本适用场景

- ✅ "上游组件暴露只读配置,下游组件内部使用同一份配置"
- ✅ "dataclass/frozen snapshot 共享于两个组件"
- ✅ "测试断言从 `_internal_field` 迁到 `public_property`"
- ❌ 不适用:协议层/identity 层断言(`_smtp_transport is smtp_transport`)
