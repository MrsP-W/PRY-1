## 4. 关键发现

### 4.1 跨源 L2 候选触发(needs_confirm=1)

**观察**:支付宝 import 输出 `needs_confirm=1 candidate_count=1`

**含义**:支付宝的 `麦当劳(朝阳店) ¥42.00 2025-03-10` 与微信的 `麦当劳(朝阳店) ¥42.00 2025-03-10` 派生相同 `normalized_fingerprint`(沿 D6.4 transactions L2 跨源去重范本),触发跨源候选标记 `needs_confirm=True` + `candidate_match_id=微信.id`。

**业务意义**:这是 D6.4 L2 跨源去重模型的核心功能——同一商家/同金额/同日 的跨源交易自动标记为 needs_confirm,等待用户 1-click 确认(避免重复入库)。
