### 1.2 B1.1 — OutboxPriority 扩 6 类(commit 1,~50 行)

**改动**:
- `core/outbox.py:171-189` `OutboxPriority(StrEnum)` 加 3 个新枚举值:
  - `HIGH = "high"`(URGENT 之下,NORMAL 之上,30min SLA)
  - `BATCH = "batch"`(批量发送,24h SLA,可错峰)
  - `DIGEST = "digest"`(摘要合并,7d SLA)
- `_OUTBOX_PRIORITY_CHOICES` 自动更新(由 `frozenset(p.value for p in OutboxPriority)` 生成,无需手工维护)
- `_PRIORITY_SORT_KEY` 加 3 个数值映射:
  - `URGENT=5`(原 3)
  - `HIGH=4`(新)
  - `NORMAL=3`(原 2)
  - `LOW=2`(原 1)
  - `BATCH=1`(新)
  - `DIGEST=0`(新)
- 现有 `SLAEvaluator` 加 HIGH/BATCH/DIGEST 阈值(沿 sla.py L41-53 模式):
  - HIGH: threshold=30min, warning=15min
  - BATCH: threshold=24h, warning=12h
  - DIGEST: threshold=7d, warning=3d

**5 处测试**:
- `tests/core/test_outbox_priority.py`(8 cases: 6 类枚举值 + 严判白名单 + 排序稳定性)
- `tests/db/test_outbox_store.py`(加 priority="high"/"batch"/"digest" insert + by_priority 测试)
- `tests/scheduler/test_outbox_dispatcher.py`(加 HIGH/BATCH/DIGEST 排序测试)
- `tests/scheduler/test_sla_evaluator.py`(加 HIGH/BATCH/DIGEST 阈值测试)
- `tests/core/test_outbox_priority_strenum.py`(沿 D4.7.3 v1.0.5 严判范本,type 严判)

**关键决策**:
- 沿用现有 StrEnum 范本(不改 dataclass)
- 不引入新依赖(纯内置 enum + dict)
- `priority` 默认值仍是 `"normal"`(向后兼容)
- 严判白名单自动扩展(无需手工维护)

**风险 + 缓解**:
1. 优先级扩展破坏现有 SLAEvaluator → 用 HIGH/BATCH/DIGEST 阈值表预填(沿 sla.py L41-53 模式)
2. 现有 tests 引用 3 类 priority → 加新 cases 而非改旧 cases(向后兼容)
3. SLA threshold 数值差异巨大(URGENT 5min vs DIGEST 7d) → 排序键数值化(0-5)统一映射

## 4. B-5 pynput 1.7.7 + macOS Sequoia 兼容评估(P1 6/20,~1 docs-only commit)

### 4.1 现状评估(2026-06-16 上午 Spike B)

- 详见 `Agent Assistant/memory/v0.1-real-spike-b-real-2026-06-16.md`

### 4.2 评估 3 候选方案(docs-only)

**方案 A**:rumps 单进程(`pynput` 改为 `rumps.events` 自带快捷键注册)
- 优点:无新依赖,与 rumps 主循环统一
- 缺点:仅菜单栏 active 时响应快捷键,后台失灵

**方案 B**:Quartz 直接绑定(`Quartz.Framework` Python 绑定,CGEvent tap)
- 优点:macOS 原生,无 pynput 兼容问题
- 缺点:需引入 pyobjc-Quartz 依赖(增大安装包)

**方案 C**:⌥⌘N 改快捷键(换 macOS 自带快捷键如 ⌃⌥⌘N 三键)
- 优点:无新依赖
- 缺点:与系统快捷键冲突风险高
