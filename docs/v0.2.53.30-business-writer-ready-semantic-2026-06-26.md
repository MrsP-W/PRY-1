## 1. 背景与目标

**v0.2.53.27 + v0.2.53.29 已完成** env 门控自动注入 `BusinessWriterImpl` + HTML inspector 3 字段联调(`business_writer_env_enabled` / `business_writer_impl_injected` / `business_writer_ready`),但 `is_business_writer_impl_injected()` 旧版用 `is None` 判定 Impl 注入状态:

```python
# v0.2.53.27 旧版(沿 v0.2.53.18 范本)
def is_business_writer_impl_injected(self) -> bool:
    return self.business_writer is not None
```

**问题暴露**:`v0.2.53.27` 同时也允许"用户显式 `with_business_writer(BusinessWriterStub())`"的合法用法(沿 v0.2.53.18 不可变更新范本),此时 `business_writer is not None` 会**误判为已注入 Impl**,触发路径 3.5-pre env_only marker 边界错位(实际是 Stub,文案却显示"env 已开 + Impl 未注入")。

**v0.2.53.30 目标**:
1. **Impl 显式识别**:`BusinessWriterImpl.is_runtime_impl: ClassVar[bool] = True` 显式 marker,`is_business_writer_impl_injected()` 用 `getattr(writer, "is_runtime_impl", False) is True` 判定,Stub 默认 False。
2. **evaluate 保守 501**:`evaluate_writer_dry_run()` 除非 `writer_impl_injected is True` 否则保守返回 501(env_only marker 边界修复),与 `is_business_writer_impl_injected()` 同步收紧。
3. **三入口同步**:README / SESSION-STATE / MODIFICATION-LOG 顶部 v0.2.53.29 → v0.2.53.30 状态口径同步(沿 v0.2.53.25 范本)。

### 2.2 `is_business_writer_impl_injected()` 显式识别

`src/my_ai_employee/dashboard/context.py` L183-191:

```python
def is_business_writer_impl_injected(self) -> bool:
    """BusinessWriterImpl 是否已通过 `with_business_writer()` / default() 注入.

    v0.2.53.30:显式识别 `BusinessWriterImpl.is_runtime_impl`,避免 Stub 被误判为已注入。
    """
    writer = self.business_writer
    if writer is None:
        return False
    return getattr(writer, "is_runtime_impl", False) is True
```

**判定逻辑**:
| `self.business_writer` | `getattr(writer, "is_runtime_impl", False)` | 结果 |
|------------------------|---------------------------------------------|------|
| `None` | N/A | `False` |
| `BusinessWriterStub()` | 默认 False(无 marker) | `False` |
| `BusinessWriterStub` 子类(假设覆盖) | `True` | `True`(duck type 沿用) |
| `BusinessWriterImpl()` | `True`(ClassVar) | `True` |
