# v0.2.53.11 Dashboard ApprovalGate 写操作设计(2026-06-26)

> **状态**:✅ 已落地 · **范围**:POST 契约 + 默认禁写 + 审计预览 · **边界**:不写 DB / 不发 SMTP / 不写 Keychain

## 结论

新增 `POST /api/approval-gate/actions` 作为未来 Dashboard 写操作的唯一入口。当前版本只做校验、拒绝和审计预览,所有返回都保证 `write_executed=false`。

## 端点契约

| 路径 | 当前行为 |
|------|---------|
| `POST /api/approval-gate/actions` | 校验 action / target_id / env / confirm_text,默认返回拒写 |

请求示例:

```json
{
  "action": "outbox.approve",
  "target_id": 123,
  "reason": "用户点击审批",
  "dry_run": true
}
```

支持动作:

| action | 未来效果 |
|--------|---------|
| `outbox.approve` | 邮件草稿从 pending_send → approved |
| `outbox.cancel` | 邮件草稿取消 |
| `notes.confirm` | 确认 Notes 候选 |
| `finance.dismiss_anomaly` | 忽略财务异常提示 |

## 门控规则

| 条件 | 当前结果 |
|------|---------|
| 未设 `DASHBOARD_WRITE_API=1` | `403 write_disabled` |
| action / target_id 缺失或非法 | `400` |
| env 已开但缺 `confirm_text=CONFIRM_WRITE` | `403 confirmation_required` |
| env + confirm_text 齐全 | `501 write_not_implemented` |

即使最后一种情况通过契约校验,当前仍不执行真实业务写入。

## 安全边界

- 不写 DB。
- 不发 SMTP。
- 不写 Keychain。
- 不 kickstart launchd。
- 不创建/移动 tag。
- 不接真实业务 writer。
- 所有响应包含 `write_executed=false`。
