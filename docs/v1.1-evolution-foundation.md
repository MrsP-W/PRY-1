## 3. P1 能力清单（先登记，后实现）

### 3.1 能力注册表（目标字段）

每个 Agent / 数据源 / 工具能力一条记录：

| 字段 | 含义 |
|---|---|
| `id` | 稳定 ID（如 `agent.email_to_draft`） |
| `kind` | `agent` / `datasource` / `tool` / `workflow` |
| `version` | semver 或日历版本 |
| `permission` | `read_only` / `draft` / `external_write` |
| `flag` | Feature Flag 名；默认 `off` 或 `dry_run` |
| `stable` | `stable` / `labs` |
| `rollback` | 回滚动作（关 flag / 退版本 / 撤索引） |
| `eval_suite` | 绑定的评测集路径 |
| `owner` | 维护入口（模块路径） |

### 3.3 Feature Flag 设计要点（v1.1-A 实现）

- 单一注册表（建议 `src/my_ai_employee/flags/registry.py` + 本地 JSON 覆盖，**不**用远程配置）。
- 三态：`off` / `dry_run` / `on`；涉及外发的能力禁止默认 `on`。
- 每次决策写 audit：`flag_id, old, new, actor, reason, ts`。
- 回滚：关 flag ≤ 1 分钟生效；不依赖重新部署。

## 4. P1 用户反馈与离线评测集

### 4.1 反馈事件（v1.1-A）

对草稿 / 分类 / 建议记录：

- `adopt` — 用户原样采纳  
- `modify` — 用户修改后使用（保留 diff 摘要，脱敏）  
- `reject` — 拒绝 / 取消  

字段最小集：`trace_id, capability_id, decision, before_hash, after_hash?, reason_code?, created_at`。

### 4.2 评测集契约（现在落地骨架）

目录：[`tests/eval/`](../tests/eval/)

- 目标：**≥ 30** 条真实脱敏样本（邮件 / SAP 排错文案 / 新闻摘要等分类）。
- 现在：契约 + 少量合成样例；**禁止**入库真实邮箱、身份证、银行卡。
- Runner（v1.1-A）：对 prompt / 模型 / 规则变更跑回归，输出 pass rate + 分能力失败列表。

样本 JSON 字段见 [`tests/eval/SCHEMA.md`](../tests/eval/SCHEMA.md)。

## 5. P1 运行 SLO 看板（设计，P3 后实现）

复用现有 health/news journal + P3 report，增强为「每源」可见：

| 指标 | 来源 | 目标（初稿） |
|---|---|---|
| 新鲜度 | news/health `at` | health gap ≤ 15min；news gap ≤ 90min（醒机） |
| 失败率 | journal outcome | 滚动 24h 失败率可查 |
| 重试 | health `attempts` / `recovered_by_retry` | 可见 |
| 隔离条数 | P2 坏数据路径计数 | 周环比 |
| 周报 | 扩展 `p3_burn_in_report` 或独立 ops report | 每周一自动 |

**P3 期间**:继续用现有 `watch_p3_ops` / burn-in report，不改调度。
