# v0.2.72 P3-A T3 L2 — 撞坑 #91 启动链路修复

> 日期:2026-07-09
> 范围:P3-A T3 L2 / launchd digital-employee wrapper
> 性质:fix + tests + docs

## 1. 结论

本轮已按用户授权 A 修复撞坑 #91 的首选路径:
数字员工 launchd wrapper 不再执行 `~/Documents/.../ops/start-digital-employee.sh`,
而是调用部署在 `~/bin` 下的 `my-ai-employee-digital-runner`。

本轮未执行 `launchctl load -w`,未触发 SMTP,未做 Notes 生产同步,
未启用 Path4 写入,未打 v1.0 tag。

## 2. 背景

撞坑 #91 的关键证据:

```text
bash: /Users/wei/Documents/DesktopOrganizer/我的AI员工/ops/start-digital-employee.sh: Operation not permitted
```

该错误发生在 bash 执行脚本路径阶段,不是 Python 进程启动后的 TCC 弹窗。
因此本轮目标不是继续补 TCC,而是让 launchd 启动链避开 Documents 下的 `.sh` 直接执行。

## 3. 修改内容

| 文件 | 修改 |
|------|------|
| `scripts/launchd_install.sh` | 新增 `TARGET_START_RUNNER=~/bin/my-ai-employee-digital-runner`;安装时复制 `ops/start-digital-employee.sh` 到 `~/bin`;`my-ai-employee-start` 只调用 runner |
| `ops/start-digital-employee.sh` | 支持 `MY_AI_EMPLOYEE_PROJECT_ROOT` 显式项目根路径,避免复制到 `~/bin` 后误判项目目录 |
| `tests/scripts/test_launchd_install.py` | 新增 F4-F7 契约测试,锁住 runner 部署、wrapper 禁止回调 Documents、项目根路径 override、deploy-only 不进入 load |

## 4. 新启动链路

旧链路:

```text
launchd plist -> ~/bin/my-ai-employee-start -> bash <PROJECT_ROOT>/ops/start-digital-employee.sh start
```

新链路:

```text
launchd plist -> ~/bin/my-ai-employee-start -> ~/bin/my-ai-employee-digital-runner start
```

`my-ai-employee-start` 会设置:

```text
MY_AI_EMPLOYEE_PROJECT_ROOT=<PROJECT_ROOT>
```

runner 因此仍使用本项目的 `data/`,日志,菜单栏入口和 dashboard 入口。
