# 支付宝账单导出教程

> **版本**:v0.1(2026-06-15) · **承接**:D7.1-D7.6 commit 链 + D6 微信账单导出教程范本
>
> **本教程说明**:如何从支付宝 App 导出账单 CSV,供 `import_alipay.py` 解析入库。

---

## 1. 为什么需要这份教程

`my-ai-employee` v0.1 通过支付宝账单 CSV 导入所有交易记录,实现:
- 关键词规则分类(5 类:餐饮/交通/购物/居家/其他)
- 跨源去重(同日同金额同商家 → 微信+支付宝自动合并提示)
- 月度报表(后续 D8 LLM 智能分析, v0.2 启用)

## 2. 导出步骤(支付宝 App)

### 2.1 进入账单页面

1. 打开支付宝 App
2. 底部导航 **"我的"** → **"账单"**
3. 顶部右上角 **"..."** → **"开具交易流水证明"**

### 2.2 选择时间范围

- 支付宝账单支持**按月**导出(2024/2025/2026 三版字段差异)
- 时间范围建议 1-3 个月(避免单文件过大)
- 注意:**单次最多 1 年**,超出需多次导出

### 2.3 选择导出格式

- **CSV 格式**(必须,不要选 PDF)
- 接收邮箱:推荐使用你的常用邮箱(QQ/163/Gmail)
- 点击 **"申请导出"**

### 2.4 邮箱接收 + 解压

- 支付宝会在 5-10 分钟内发送邮件
- 邮件附件为 **ZIP 压缩包**(内含 CSV)
- 解压后得到 `alipay_record_YYYY-MM-DD.csv`(文件名格式不固定)

## 3. 文件命名建议

为方便 `import_all.py` 自动嗅探,建议将文件重命名为:
- `alipay_2024_sample.csv`
- `alipay_2025_sample.csv`
- `alipay_2026_sample.csv`

统一放在一个目录,例如 `~/Downloads/bills/`,然后:

```bash
uv run python scripts/import_all.py --csv-dir ~/Downloads/bills
```

## 6. 常见问题

### Q1:CSV 打开乱码?

A:支付宝导出 CSV 是 UTF-8 BOM 编码,Excel 默认按 GBK 解析会乱码。
**解决**:
- Mac:用 Numbers 打开
- Windows:用记事本打开后另存为 UTF-8
- 或直接用 VSCode 打开(自动识别 BOM)

### Q2:跨源同 fingerprint 提示 needs_confirm?

A:微信和支付宝同日同金额同商家会被自动标记为"待确认"。
**原因**:可能是同一笔消费(微信支付 + 支付宝分摊)或两次重复消费。
**处理**:在 `transactions` 表查 `needs_confirm=1` 的记录,人工 review 决定合并/保留。

### Q3:Alembic 校验失败?

A:DB 没跑 `0007_transactions` migration。
**解决**:
```bash
uv run alembic upgrade head
```

### Q4:导入 0 行?

A:CSV header 没匹配 2024/2025/2026 任一版本(可能是 2018-2023 旧版)。
**解决**:
- 升级支付宝 App 到最新版重导
- 或在 D7.1 加新版本 parser(沿 plan §3 D7.1 范本)

## 7. D7 跨源去重机制

D7 关键创新:**微信 + 支付宝跨源同 fingerprint 自动去重**。

例如:
- 2026-06-14 12:00 微信支付星巴克 38.50
- 2026-06-14 12:00 支付宝花呗星巴克 38.50

→ L2 跨源候选命中 → 新交易标记 `needs_confirm=1` + `candidate_match_id=老交易id`

**绝不**自动 delete/update 老交易(防误合并)。
