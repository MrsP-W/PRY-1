## 2. 设计 — 报告分类 + 元数据提取

### 2.1 报告类型(4 类)

| type | 来源 | 典型文件 |
|------|------|----------|
| `doc` | `docs/*.md` | `docs/v0.2.53.7-dashboard-opt-in-real-db-2026-06-25.md` |
| `phase_report` | `reports/*.md` | `reports/D-脚手架完成.md` · `reports/2026-07-monthly-review.md` |
| `spike` | `output/spike*/` | `output/spike/spike_send_100_20260625_163607.md` |
| `agent_output` | `output/YYYY-MM-DD/` | `output/2026-06-17/端午连休准备.md` |

### 2.2 元数据提取(沿 v0.2.53.4 safe_* 范本)

| 字段 | 提取方式 | 边界 |
|------|----------|------|
| `path` | `path.relative_to(root).as_posix()` | POSIX 相对路径,无绝对前缀 |
| `type` | 目录归属判定 | 4 类枚举 |
| `title` | 读前 5 行找第一行 `#` 去掉 `# + strip()` | 无 H1 fallback 文件名 stem |
| `date` | 文件名 regex 解析(2 种 pattern:YYYY-MM-DD / YYYYMMDD) | 无效日期 → 空字符串 |
| `status` | 扫前 30 行匹配关键词(`✅`/`🟢`/`🟡`/`❌`/`📝`)| `done`/`active`/`pending`/`failed`/`draft`/`unknown` |
| `size_bytes` | `path.stat().st_size` | OSError → 0 |

### 2.3 响应格式(`build_reports_payload`)

```json
{
  "read_only": true,
  "count": 42,
  "items": [
    {
      "path": "docs/v0.2.53.7-dashboard-opt-in-real-db-2026-06-25.md",
      "type": "doc",
      "title": "v0.2.53.7 Dashboard opt-in 真实 Outbox",
      "date": "2026-06-25",
      "status": "done",
      "size_bytes": 6020
    }
  ]
}
```

**支持查询参数**:
- `?limit=N` — 最大返回条数(默认 50,严判 [1,100])
- `?type=<doc|phase_report|spike|agent_output>` — 仅返回指定类型

## 3. 实现要点

### 3.1 扫描器入口(`reports.py`)

```python
def scan_reports(
    *,
    project_root: Path | None = None,
    limit: int = 50,
    type_filter: str | None = None,
) -> list[ReportEntry]:
    if limit < 1:
        limit = 1
    root = project_root or Path.cwd()
    try:
        entries: list[ReportEntry] = []
        entries.extend(_scan_docs(root))
        entries.extend(_scan_reports(root))
        entries.extend(_scan_output(root))
        if type_filter:
            entries = [e for e in entries if e.type == type_filter]
        entries.sort(key=_sort_key, reverse=True)
        return entries[:limit]
    except Exception:
        return []  # 静默降级,API 层不崩
```

### 3.2 handler 路由(`handlers.py`)

```python
if path == "/api/reports":
    self._send_json(
        HTTPStatus.OK,
        build_reports_payload(
            self.dashboard_context, limit=limit, type_filter=type_filter
        ),
    )
    return
```

## 5. 撞坑 #66(本轮新增)— 报告扫描器 5 不做原则

**范本**:任何"把本地文件变成 Dashboard 报告"场景:
1. **不接真实外部服务**(无 HTTP 拉取 / 无数据库查询 / 无 Keychain 读取)
2. **不返回邮件 body / 敏感字段**(只返回 6 字段元数据)
3. **不写操作**(只读文件系统,不创建 / 删除 / 修改任何文件)
4. **不限制文件大小读取**(只读前 5 行找标题 + 前 30 行找状态,避免 OOM)
5. **不引入新依赖**(纯 Python stdlib: pathlib + re + dataclasses)

### 5.1 范本适用场景

- ✅ 把已有 docs/ + reports/ + output/ 沉淀变成可浏览资产
- ✅ 任何"本地文件清单 → Dashboard 列表"场景
- ✅ 文件名带日期 / 状态关键词的纯文件型报告
- ❌ 不适用:需要实时统计文件内容(如行数 / 字符数)— 留给 v0.2.53.x 后续
