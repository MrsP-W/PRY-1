## 4. 撞坑观察与教训

### 4.1 撞坑 #36(本轮新增) — 跨源构造必须严格同符号

**现象**:v0.2.26 spike 第一版构造时,跨源 pair 的 `收/付` vs `收/支` 用了随机 direction,导致跨源 amount 符号不一致。

**根因**:微信 `付` → `-amount`,支付宝对应语义 `支` → `+amount`(+ 号省略);微信 `收` → `+amount`,支付宝对应 `收` → `-amount`。第一版 random.choice 破坏了这个对应。

**指纹分析**:虽然 `normalize_fingerprint` 用 `abs(amount)` 仍然会命中,但 amount 语义不一致会导致:
1. 财务对账错误(微信支出 vs 支付宝收入)
2. 用户看到 1 个"负" + 1 个"正"的同源记录困惑
3. `categorized` vs `needs_confirm` 状态机的语义混乱

**修法**:cross-source pair 构造时显式锁 `sign = ±1`,微信 `付`/`收` + 支付宝 `支`/`收` 共用同一 sign。

**防坑**:
- 任何"跨源同金额"测试必须 lock sign(不只是 lock amount)
- 双源 direction 映射表(微信 `付` ↔ 支付宝 `支` / 微信 `收` ↔ 支付宝 `收`)要写在 spike 脚本注释里
- 后续生成器模块化时,`CrossSourcePairGenerator` 强制 sign 一致

### 4.2 撞坑 #21(本轮再触发) — pwd 漂移

**现象**:`/tmp/spike_w3_virtual.py` 跑 `merchants.py` 找不到 `tests/fixtures/merchants_500.json`。

**根因**:cwd 是 `/tmp/`,merchants.py 用相对路径 `_DEFAULT_MERCHANTS_PATH = Path("tests/fixtures/merchants_500.json")`,相对 cwd 解析失败。

**修法**:`os.chdir(PROJECT_ROOT)` 切到项目根(沿 v0.2.18 §3 撞坑史 6 类高频撞坑)

**防坑**:
- 任何 spike 脚本都用 `os.chdir(PROJECT_ROOT)` 兜底
- 或改 `_DEFAULT_MERCHANTS_PATH = Path(__file__).resolve().parents[2] / "tests" / "fixtures" / "merchants_500.json"`(更彻底,本次不修)

### 4.3 撞坑 #11(沿 v0.2.18 §3) — Faker 数据命名

**现象**:生成器临时 CSV 放在 `/tmp/`,命名 `spike_w3_virtual_*`(避开 `faker` 命名)。

**根因**:沿 v0.2.1 #2 virtual spike 范本,避免被 `_validate_csv` 防 faker 误传检查拦截。

**沿用**:命名 `*virtual*` 或 `*spike*`,绝不叫 `*faker*`。
