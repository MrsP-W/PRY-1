## 1. 一句话总结

`OutboxDispatcher` 构造后**自动从 adapter 读取 `_provider_default_*` 3 字段**(沿 v0.2.52 #61 暴露字段),运行时优先级 = provider 默认值(非 None) > 显式 smtp_host/port/username(构造时传),冲突时严判 ValueError(沿撞坑 #18 范本防 silent override)。

---

## 2. 设计 — 5 路径严判(撞坑 #18 范本)

| # | 路径 | 判定 | 行为 |
|---|------|------|------|
| **1** | **provider + 默认值可用** | `adapter._smtp_provider` 非 None + `_provider_default_host/port/email` 全非 None | 优先级最高:OutboxDispatcher 同步 3 字段,运行时用 `_provider_default_*` 覆盖显式字段(无需显式 `smtp_host/port/username/password`)|
| **2** | **provider + 默认值缺失**(edge)| provider 非 None 但 `_provider_default_host is None`(沿 v0.2.52 None fallback)| fallback 路径 3 显式字段 |
| **3** | **显式 smtp_host/port/username/password**(传统路径)| OutboxDispatcher 构造时传完整 4 字段 | 用 OutboxDispatcher 自带的 4 字段(provider 模式可共存)|
| **4** | **完全没传 SMTP 配置**(测试默认)| 全 None / 占位默认值 | 拒绝启动(ValueError 严判),沿 D5.6.1 P0 范本 |
| **5** | **优先级冲突**(provider 默认值 vs 显式传入)| provider 非 None + 显式 `smtp_host` 与 `_provider_default_host` 不一致 | **严判冲突 + ValueError**(防止 silent override)|

## 3. 实现要点

### 3.1 同步 3 字段(`__init__` 末尾)

`src/my_ai_employee/scheduler/outbox_dispatcher.py:336-379` 新增 `4.5` 段:

```python
# v0.2.52.1 OutboxDispatcher 自动路由(provider 模式):
self._provider_default_host: str | None = None
self._provider_default_port: int | None = None
self._provider_default_email: str | None = None
self._active_provider: str | None = None
if send_adapter is not None:
    adapter_provider = getattr(send_adapter, "_smtp_provider", None)
    if adapter_provider is not None:
        self._active_provider = adapter_provider
        self._provider_default_host = getattr(send_adapter, "_provider_default_host", None)
        self._provider_default_port = getattr(send_adapter, "_provider_default_port", None)
        self._provider_default_email = getattr(send_adapter, "_provider_default_email", None)
        # 严判冲突:provider 默认 host/port 与显式 smtp_host/port 不一致 → ValueError
        # 沿撞坑 #18 范本 — 严判不静默
        if (
            self._provider_default_host is not None
            and smtp_host != "smtp.test.local"
            and smtp_host != self._provider_default_host
        ):
            raise ValueError(
                f"smtp_host 与 provider 默认 host 冲突: smtp_host={smtp_host!r} "
                f"provider_default_host={self._provider_default_host!r}"
            )
        # ... smtp_port 类似
```

### 3.2 运行时优先级(`_process_one_entry`)

- **From 字段**:`adapter._provider_default_email`(非 None) > `self._smtp_username`(fallback)
- **smtp_host/port**:`adapter._provider_default_host/port`(非 None) > `self._smtp_host/port`(fallback)
- **smtp_username/password**:**不覆盖**(沿用 OutboxDispatcher 构造时显式字段)

### 3.3 冲突严判

```python
if (
    self._provider_default_host is not None
    and smtp_host != "smtp.test.local"  # 占位默认值 → 跳过严判
    and smtp_host != self._provider_default_host  # 不一致 → ValueError
):
    raise ValueError(...)
```

**关键点**:`smtp_host="smtp.test.local"` 是构造默认值(单元测试用),跳过严判;只有显式传非默认值时才严判冲突。
