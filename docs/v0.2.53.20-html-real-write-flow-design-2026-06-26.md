## 二、实际写入 HTML 流程(目标态设计)

### 2.1 Mail · pending_send 实写审批

```
1. 用户点击「审批」按钮(outbox_id=123)
2. HTML postJson('/api/approval-gate/actions', {action: 'outbox.approve', target_id: '123', reason: '用户点击审批'})
3. handler 评估:
   - DASHBOARD_WRITE_API=1 ✓
   - confirm_text=CONFIRM_WRITE ✓
   - BUSINESS_WRITER_ENABLED=1 ✓
   - writer 已注入 ✓
4. writer.approve_outbox('123', audit=AuditContext(actor='local_dashboard', reason='用户点击审批'))
5. OutboxStore.update_status(123, APPROVED, from_status=PENDING_SEND, last_approved_at_ms=now_ms)
6. 审计落档 approval_gate_audits 表
7. 响应: HTTP 200 + {write_executed: true, affected_id: '123', audit_id: 456, ...}
8. HTML inspector 展示「✅ 已审批(outbox 123, audit 456)」
```

## 三、audit log 表设计(0016 alembic migration · 留 v0.2.53.20+ 实施)

### 3.1 approval_gate_audits 表

```sql
CREATE TABLE approval_gate_audits (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    action TEXT NOT NULL,                  -- 4 类之一(outbox.approve 等)
    target_id TEXT NOT NULL,                -- str(沿契约)
    actor TEXT NOT NULL DEFAULT 'local_dashboard',
    reason TEXT NOT NULL DEFAULT '',
    write_executed INTEGER NOT NULL,        -- 0/1 BOOLEAN 走 Integer(沿 D3.2 #2)
    affected_id TEXT NULL,                  -- 实际写入后 ID
    error TEXT NULL,                        -- 失败原因
    executed_at_ms INTEGER NOT NULL,
    UNIQUE(action, target_id, executed_at_ms)  -- 防止同一时刻同一 target 重复落档
);
CREATE INDEX idx_executed_at ON approval_gate_audits (executed_at_ms DESC);
```

### 3.2 ORM 模型

```python
class ApprovalGateAudit(Base):
    """ApprovalGate 实际写入审计表(mirror 0016 alembic migration)."""

    __tablename__ = "approval_gate_audits"

    id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
    action: Mapped[str] = mapped_column(Text, nullable=False)
    target_id: Mapped[str] = mapped_column(Text, nullable=False)
    actor: Mapped[str] = mapped_column(Text, nullable=False, server_default="local_dashboard")
    reason: Mapped[str] = mapped_column(Text, nullable=False, server_default="")
    write_executed: Mapped[int] = mapped_column(Integer, nullable=False)  # 0/1 BOOLEAN
    affected_id: Mapped[str | None] = mapped_column(Text, nullable=True)
    error: Mapped[str | None] = mapped_column(Text, nullable=True)
    executed_at_ms: Mapped[int] = mapped_column(Integer, nullable=False)

    __table_args__ = (
        UniqueConstraint("action", "target_id", "executed_at_ms", name="uq_audit_action_target_time"),
        Index("idx_executed_at", text("executed_at_ms DESC")),
    )
```

## 五、离线兜底机制(沿 v0.2.53.12)

```javascript
async function postJson(path, body) {
    try {
        const response = await fetch(path, {
            method: 'POST',
            headers: {'Content-Type': 'application/json'},
            body: JSON.stringify(body),
        });
        if (!response.ok) {
            return {error: 'http_error', status: response.status};
        }
        return await response.json();
    } catch (e) {
        // API 离线兜底(沿 v0.2.53.12)
        return {error: 'api_offline', write_executed: false};
    }
}
```

**关键不变式**:
- API 离线时 `write_executed: false` 恒定
- 不发起 fetch 不会写入
- inspector 展示「API 离线」
