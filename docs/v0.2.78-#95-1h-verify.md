## 🚨 撞坑 #97 新暴露 — SQLCipher 跨线程 close 报错

### 现象(dashboard.err.log 截取)

```text
Exception closing connection <sqlcipher3.dbapi2.Connection object at 0x1069f4b30>
Traceback (most recent call last):
  File "/.../sqlalchemy/pool/base.py", line 375, in _close_connection
    self._dialect.do_close(connection)
  File "/.../sqlalchemy/engine/default.py", line 721, in do_close
    dbapi_connection.close()
sqlcipher3.dbapi2.ProgrammingError: SQLite objects created in a thread can only
be used in that same thread. The object was created in thread id 6232059904
and this is thread id 6248886272.
```

### 根因分析

- SQLCipher 默认 `check_same_thread=True`(与 sqlite3 默认一致)
- Dashboard server 每个 HTTP 请求 spawn 新 thread
- connection 在 thread 0 创建,close 时跑到 thread 1 触发 thread check 失败
- 这是 SQLAlchemy pool GC 阶段抛的,不影响 request-time
- 错误仅 stderr 累积(0 ERROR/CRITICAL,12 INFO 类别,但 traceback 仍多行)

### 严重度评估

| 维度 | 评估 |
|------|------|
| 是否 fatal | ❌ 否(close-time 报错,request-time 正常) |
| 是否让进程退出 | ❌ 否(Dashboard PID 11406 持续 1h 零重启) |
| 是否阻塞 HTTP 请求 | ❌ 否(404 4ms 响应) |
| err log 累积速度 | ⚠️ 30→60min +38 行(每请求 +1 close-time traceback) |
| 是否需要立刻修 | ⚠️ **建议 P0-4 24h 观察前修**(避免 24h 累积数千行 traceback) |
| 修复候选 | 1) `check_same_thread=False` + StaticPool<br>2) `scoped_session`(thread-local)<br>3) `event.listens_for(engine, "connect")` 设 PRAGMA |

### 临时缓解(可选)

如不修,24h 观察可能累积 ~3000 行 close-time traceback,但服务仍可用。

## 🔧 撞坑 #97 修复路径(待 P0-4 24h 观察前决策)

### 路径 A:`check_same_thread=False` + StaticPool(最快)

```python
engine = create_engine(
    f"sqlite:///{db_path}?check_same_thread=False",
    connect_args={"check_same_thread": False},
    poolclass=StaticPool,
)
```

- ✅ 改动小(单文件,~10 行)
- ✅ 立即解决
- ⚠️ StaticPool 单 connection 多 thread 共享,需 ensure SQLCipher 主键/连接安全

### 路径 B:`scoped_session`(thread-local,推荐)

```python
from sqlalchemy.orm import scoped_session, sessionmaker
Session = scoped_session(sessionmaker(bind=engine))
# 每次 thread 自动取 thread-local session
```

- ✅ 范本最标准(SQLAlchemy 官方推荐)
- ✅ 改动中等(~30 行,需重写所有 DB 调用点)
- ⚠️ 需小心 session lifecycle(避免 leak)

### 路径 C:`event.listens_for(engine, "connect")` PRAGMA

```python
@event.listens_for(engine, "connect")
def set_sqlite_pragma(dbapi_connection, connection_record):
    cursor = dbapi_connection.cursor()
    cursor.execute("PRAGMA foreign_keys=ON")
    cursor.close()
```

- ✅ 改动小(单文件)
- ⚠️ **不直接修 #97 跨线程问题**,需配合 A 或 B

### 推荐:**路径 A**(最小改动,先观察 24h 验证)
