# v0.2.74 — P3-A T3 L2 撞坑 #92 修复路径 B(path migration · 2026-07-09)

> **一句话**:把 launchd 运行时配置 / 日志 / 数据路径迁出 `~/Documents/`(iCloud 同步目录沙箱),命中 macOS OS 层拦截。撞坑 #92 与 #91 同根因(`bash` exec / `grep` / `>` redirect 路径遇 Documents 沙箱触发 `Operation not permitted`),撞坑 #91 修了 exec 阶段,撞坑 #92 暴露业务代码层同一根因。

---

## 🎯 撞坑 #92 — 业务代码路径撞 Documents 沙箱

### 现象

`launchctl load -w com.myaiemployee.digital-employee.plist` 后,business code 在 Documents 路径继续撞沙箱:

```text
grep: /Users/wei/Documents/DesktopOrganizer/我的AI员工/.env: Operation not permitted
/Users/wei/bin/my-ai-employee-digital-runner: line 252: /Users/wei/Documents/.../data/menu_bar.log: Operation not permitted
```

**9 维度预检 fail 链**(因 `.env` 读不到):5/9 OK + 4/9 fail + 9/9 菜单栏启动失败。

### 根因

macOS `~/Documents/` 是 iCloud 同步目录,任何 `bash exec` / `grep` / `> redirect` 操作都会被 OS 层沙箱拦截。
- 撞坑 #91 修了 launchd exec 阶段(已改 wrapper 调 `~/bin/`)
- 撞坑 #92 是同一根因在 runtime 业务代码层的延续

### 修复路径 B — 全部 runtime 路径迁 `~/Library/Application Support/`

沿 [[v1.0-launch-runbook]] 范本:

| runtime 用途 | 旧路径(Documents 沙箱) | 新路径(非 Documents) |
|------|------|------|
| .env | `$PROJECT_ROOT/.env` | `$HOME/Library/Application Support/MyAIEmployee/.env` |
| data/ | `$PROJECT_ROOT/data/` | `$HOME/Library/Application Support/MyAIEmployee/data/` |
| PID 文件 | `$PROJECT_ROOT/data/menu_bar.pid` | `$APP_SUPPORT_DIR/data/menu_bar.pid` |
| 业务日志 | `$DATA_DIR/logs/*.log` | `$HOME/Library/Logs/MyAIEmployee/*.log` |

## 🛠️ 实施细节

### 1. `scripts/launchd_install.sh` 改 heredoc + APP_SUPPORT_DIR setup

- 新增 `APP_SUPPORT_DIR="${HOME}/Library/Application Support/MyAIEmployee"` + `APP_SUPPORT_ENV="${APP_SUPPORT_DIR}/.env"`
- 段落「`# ===== 2. 目标目录校验 + 自动创建 =====`」后追加 APP_SUPPORT_DIR 创建 + 权限校验 + .env 自动迁移(若 `${PROJECT_ROOT}/.env` 存在且 `${APP_SUPPORT_ENV}` 不存在 → `cp` 迁移)
- IMAP wrapper / start wrapper 生成改 heredoc(`cat << EOF`)替代旧 `echo "..."` 模式(避免 `\\"` 转义地狱,且让 F5/F9/F10 测试断言可直接命中)
- digital-runner wrapper 显式 export 三个变量:`MY_AI_EMPLOYEE_PROJECT_ROOT` + `MY_AI_EMPLOYEE_APP_SUPPORT_DIR` + `MY_AI_EMPLOYEE_ENV_FILE`

### 2. `ops/start-digital-employee.sh` 路径变量迁 APP_SUPPORT

- `APP_SUPPORT_DIR="${MY_AI_EMPLOYEE_APP_SUPPORT_DIR:-$HOME/Library/Application Support/MyAIEmployee}"` 替代 `$PROJECT_ROOT/data`
- `ENV_FILE="${MY_AI_EMPLOYEE_ENV_FILE:-$APP_SUPPORT_DIR/.env"` 替代 `$PROJECT_ROOT/.env`
- `DATA_DIR="$APP_SUPPORT_DIR/data"`(撞坑 #92 修复)
- `LOG_DIR="${MY_AI_EMPLOYEE_LOG_DIR:-$HOME/Library/Logs/MyAIEmployee}"`(沿 v1.0 launch plan 范本,本就是非 Documents)
- `_get_imap_user_from_env()` 读 `$ENV_FILE`
- 预检 1/9 .env 检查读 `$ENV_FILE`、2/9 DB_ENCRYPTION_KEY 检查读 `$ENV_FILE`、9/9 data/ 检查读 `$DATA_DIR`

### 3. `tests/scripts/test_launchd_install.py` 新增 5 个测试

- **F8**:APP_SUPPORT_DIR 创建 + `.env` 自动迁移(撞坑 #92 修复)
- **F9**:digital-runner wrapper export `MY_AI_EMPLOYEE_APP_SUPPORT_DIR` + `MY_AI_EMPLOYEE_ENV_FILE`
- **F10**:IMAP wrapper `ENV_FILE="${APP_SUPPORT_ENV}"` 而非 `${PROJECT_ROOT}/.env`
- **G1**:`start-digital-employee.sh` 读 `ENV_FILE` 而非 `$PROJECT_ROOT/.env`(且 grep 不直接读 Documents/.env)
- **G2**:`start-digital-employee.sh` `DATA_DIR="$APP_SUPPORT_DIR/data"` 而非 `$PROJECT_ROOT/data`

F5 同步调整:`assert 'export MY_AI_EMPLOYEE_PROJECT_ROOT="${PROJECT_ROOT}"'`(heredoc 无 `\\"` 转义,旧 echo 形式已下线)。

#!/usr/bin/env bash
# 部署于 2026-07-09 14:18:23 by scripts/launchd_install.sh
# 撞坑 #92 修复(2026-07-09):ENV_FILE 路径从 PROJECT_ROOT/.env → APP_SUPPORT/.env
ENV_FILE="/Users/wei/Library/Application Support/MyAIEmployee/.env"

  IMAP_USER=$(grep -E '^IMAP_USER=' "${ENV_FILE}" | head -1 | cut -d= -f2- | tr -d '"' | tr -d "'")

  echo 'IMAP_USER not set in .env' >&2
  exit 2
fi
