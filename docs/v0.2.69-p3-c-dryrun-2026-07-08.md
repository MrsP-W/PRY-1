### 撞坑 #88 修复路径(从撞红到全绿)

#### 首跑 — 撞坑 #88 Layer 1(`update_status` 缺 `from_status` 关键字)

```
TypeError: OutboxStore.update_status() missing 1 required keyword-only argument: 'from_status'
```

**根因**: D5.2 状态机严判 — `update_status` 加 `from_status` 必传关键字(防 concurrent 写漂移)+ `last_approved_at_ms` 必传规则(APPROVED 写,其他 None 保留)。

#### 二次跑 — 撞坑 #88 Layer 2(状态机非法跳:`APPROVED → SENT` 不在白名单)

```
OutboxIllegalTransitionError: outbox_id=1 状态机非法转换: 'approved' → 'sent'
(allowed from 'approved': ['cancelled', 'failed', 'sending'])
```

**根因**: D5.2 ALLOWED_TRANSITIONS 6 状态机:`APPROVED → SENDING` 是 SMTP 调度必经中间态,`APPROVED → SENT` 直跳非法。

#### 最终修复(两层一起改)

```python
# 撞坑 #88 修复后 — scripts/spike_outbox_100.py 状态机段
print("   状态机验证:100 行 pending_send → approved → sending → sent ...")
now_ms = int(time.time() * 1000)
for outbox_id in outbox_ids:
    # 5a. PENDING_SEND → APPROVED(D5.6.3 P1-1 写审批凭据)
    row = outbox_store.update_status(
        outbox_id, OutboxStatus.APPROVED.value,
        from_status=OutboxStatus.PENDING_SEND.value,
        last_approved_at_ms=now_ms,
    )
    # 5b. APPROVED → SENDING(SENT 必经中间态 · D5 调度器触发)
    row1 = outbox_store.update_status(
        outbox_id, OutboxStatus.SENDING.value,
        from_status=OutboxStatus.APPROVED.value,
        last_approved_at_ms=None,
    )
    # 5c. SENDING → SENT(spike 0 SMTP,仅状态推进模拟)
    row2 = outbox_store.update_status(
        outbox_id, OutboxStatus.SENT.value,
        from_status=OutboxStatus.SENDING.value,
        last_approved_at_ms=None,
    )
```

**改动范围**:
- 1 处(状态机 block)· OLD 5 行 → NEW 30 行(含 3 段注释 + 3 跳 state machine)
- 文件:`scripts/spike_outbox_100.py` line 266-289
- 改动行数符合"scripts/ ≠ src/ 业务代码"边界(撞坑 #71 严格遵守)
