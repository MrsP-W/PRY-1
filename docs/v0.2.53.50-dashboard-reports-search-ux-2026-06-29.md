## 1. 背景与目标

v0.2.53.9 + v0.2.53.10 落地后,报告页已具备基础能力:`/api/reports` 列表 + `/api/reports/preview` 详情 + filter(全部/docs/phase_report/spike/agent_output)+ 标题/路径搜索 + 点击预览。但 UX 仍存在 4 类可强化点:

- **排序**:默认按 `_currentReports` 顺序(沿文件系统遍历顺序,无日期概念),用户找最新报告需肉眼扫
- **搜索体验**:搜索词命中时无视觉反馈,难定位匹配片段;无清除按钮
- **匹配计数**:无"X / Y 项匹配"实时反馈,用户不知道搜索是过严还是过宽
- **空状态**:搜索无结果时只显示"暂无匹配报告",未提示搜索词

v0.2.53.50 目标(只读 GET,纯前端 UX 强化,不动后端契约):

- 按日期倒序(最近优先),null/空日期排末尾
- 搜索词高亮(标题 + 路径命中片段 `<mark>` 标签)
- 清除按钮(显示/隐藏 + Escape 键兜底)
- 实时匹配计数(`X / Y 项匹配「query」`)
- 搜索无结果空状态展示搜索词(`<mark>` 高亮)

## 2. 设计决策(2026-06-29 锁定)

### 2.1 报告排序(日期倒序)

```javascript
function filteredReports() {
  // ... 过滤逻辑 ...
  return out.sort((a, b) => {
    const da = String(a.date ?? "");
    const db = String(b.date ?? "");
    if (!da && !db) return 0;
    if (!da) return 1;
    if (!db) return -1;
    return db.localeCompare(da);  // 倒序:最近优先
  });
}
```

**排序语义表**:

| 报告 A 日期 | 报告 B 日期 | 排序结果 |
|------------|------------|---------|
| "2026-06-29" | "2026-06-28" | A 排前(降序) |
| "2026-06-29" | null | A 排前(有日期优先) |
| null | null | 保持原顺序 |
| null | "2026-06-28" | B 排前(null 排末尾) |

**`localeCompare` 选择**:日期字符串 ISO 8601 格式(`YYYY-MM-DD`)字典序 = 时间序,无需 `new Date()` 解析(避免跨时区歧义 + 性能)。

### 2.2 搜索词高亮(`<mark>` 标签)

```javascript
function highlightMatch(text, query) {
  if (!text || !query) return escapeHtml(text || "");
  const lowerText = String(text).toLowerCase();
  const lowerQuery = query.toLowerCase();
  const idx = lowerText.indexOf(lowerQuery);
  if (idx < 0) return escapeHtml(text);
  const before = text.slice(0, idx);
  const hit = text.slice(idx, idx + query.length);
  const after = text.slice(idx + query.length);
  return `${escapeHtml(before)}<mark>${escapeHtml(hit)}</mark>${escapeHtml(after)}`;
}
```

**应用位置**:
- 报告标题(`.queue-title`):`highlightMatch(r.title, query)`
- 报告路径(`.queue-meta.mono`):`highlightMatch(r.path, query)`
- 空状态提示:`未找到匹配「<mark>query</mark>」的报告`

**安全要点**:
- 所有片段经 `escapeHtml` 转义后拼接(沿撞坑 #50 范本)
- `query` 作为纯字符串插入 `<mark>` 内容,无 XSS 风险(浏览器解析 `query` 为文本)

### 2.3 清除按钮 + Escape 兜底

```html
<button id="reports-search-clear" class="filter" type="button" hidden>清除</button>
```

**交互逻辑**:
- 输入框 `input` 事件:`query.length === 0` 时隐藏清除按钮,否则显示
- Escape 键:清空输入 + 隐藏按钮 + focus 输入框
- 清除按钮 click:同 Escape 行为

**沿用范本**:`<button type="button">` 防止 form submit · `hidden` 属性原生支持(无需 CSS) · `aria-label="清除搜索"` 提升无障碍。

### 2.4 实时匹配计数

```html
<span class="tag" id="reports-match-count">0 项</span>
```

**显示规则**:

| 场景 | 文案 |
|------|------|
| 无搜索 + filter=all | `${total} 项` |
| 无搜索 + filter=X | `${filtered.length} / ${total} 项 (${X})` |
| 有搜索 | `${filtered.length} / ${total} 项匹配「${query}」` |

**4 状态映射**:与 `filteredReports()` 同步更新,每次 render 重渲染。
