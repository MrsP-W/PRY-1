## D8 — 智能财务分析

### 目标

LLM 驱动的异常检测 + 商家分类补充 + 月度统计。

### 任务清单

| # | 任务 | 预计耗时 | 产出 |
|---|------|----------|------|
| 8.1 | 写 `ai/finance_analyzer.py`（minimax M3）| 90 min | 分析服务 |
| 8.2 | 写 `ai/prompts/finance_anomaly.txt` | 30 min | 异常检测 prompt |
| 8.3 | 写 `ai/prompts/finance_categorize.txt` | 30 min | 分类补充 prompt |
| 8.4 | 写 `core/budget.py`（预算配置 + 告警）| 60 min | 预算引擎 |
| 8.5 | 写 `scripts/detect_anomaly.py`（异常交易扫描）| 30 min | 异常入口 |
| 8.6 | **Spike**：500 笔手标做 LLM vs 规则对比 | 90 min | 准确率报告 |
| 8.7 | 写 `tests/ai/test_finance_analyzer.py` | 60 min | 单元测试 |

**总耗时**：约 6.5 小时

### 验收标准

- [ ] LLM 分类补充后整体准确率 ≥ 90%
- [ ] 异常检测：单笔 > 1000 元 + 偏离均值 3σ 触发告警
- [ ] 预算超支 80% 触发提醒
- [ ] 单笔分析 < 5s
- [ ] LLM 不可用时降级到规则引擎（统计 + 阈值）

### 风险点

- **异常检测基线**：需要 ≥ 30 天数据才能算 σ
- **LLM 成本**：500 笔 × minimax M3 估算成本
- **隐私**：交易含商家名，部分敏感（如医院）需脱敏（跳过 LLM）

### 📌 下一棒 → D9

- 财务智能层就绪
- 下棒需要：1 个月历史交易数据
- 关键决策：月报用模板化生成还是 LLM 自由发挥？— 我推荐**模板化**（稳定 + 可控）
