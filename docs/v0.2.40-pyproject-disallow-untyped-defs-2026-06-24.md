### 4.2 撞坑 #56(本轮新增)— 批量脚本 AST 注入顺序陷阱

**现象**:批量脚本 `add_any_annotations.py` 把 `from typing import Any` 插在 `from __future__ import annotations` 前面 → Python SyntaxError(因 `from __future__` 必须是文件首行)。

**根因**:脚本检测"已有 typing import"时,只查找 `from typing import` 行,未考虑 Python 语言规范要求 `from __future__` 必须在最前。

**范本**:**批量 AST 注入顺序陷阱** — 任何"批量加 import"操作必须先做"future import 检查",**保证 `from __future__` 永远在最前**。

**教训**:**自动化脚本必须处理 Python 语言层强约束**(`from __future__` 顺序)而不仅是业务层逻辑。
