### 1.2 v2 新增核心:显式 launch flag

**C3 没有显式 launch flag**,v0.2.53.49 实施后,`_real_write_handler_enabled` 仅是构造参数(默认 False)。
v2 引入顶级 env `ENABLE_PATH_4_WRITE=1`,作为**第 5 道门**的显式开关:

```bash
# 8/1 后启动路径 4 实写(必须 4 门 + 5th flag 全部就位)
ENABLE_PATH_4_WRITE=1 \
DASHBOARD_WRITE_API=1 \
BUSINESS_WRITER_ENABLED=1 \
DASHBOARD_REAL_DB=1 \
make dashboard-api
```

**设计意图(沿撞坑 #18 风险门控 强化)**:
- `ENABLE_PATH_4_WRITE=1` 是**第 5 道门**(在 BusinessWriterImpl 内部严判)
- 未设此 env 即便其他 4 门都开,BusinessWriterImpl._call_service_xxx 也 raise NotImplementedError
- 这是"二次确认"机制,避免运维误启导致 SMTP/DB/Keychain 真实写

### 1.3 v2 新增核心:实施回滚 plan

**C3 的回滚 plan**:任何步骤失败 → 回滚到 `v0.2.53.42` 状态(`73552a3` HEAD)

**v2 增强回滚 plan**(沿撞坑 #18「日志」语义):
- 4 门任一未达 → 立即 `raise NotImplementedError`(沿 v0.2.53.46 默认行为)
- `ENABLE_PATH_4_WRITE` 未设 → 立即 `raise NotImplementedError`(新增)
- 真实 service 调用失败 → audit 落档失败记录 + 异常透传(沿 v0.2.53.51)
- audit_store.record() 失败 → 业务 WriteResult 仍正常返回(撞坑 #18「日志」语义)
- **永不静默成功**:任何路径下,4 门 / 5th flag / service 调用 / audit 落档 任一异常都应被记录或抛错

### 2.3 ENABLE_PATH_4_WRITE 实施位置

`BusinessWriterImpl` 构造函数新增 env 读取:

```python
import os
_ENABLE_PATH_4_WRITE_ENV = "ENABLE_PATH_4_WRITE"

def __init__(
    self,
    *,
    audit_store: ApprovalGateAuditStore | None = None,
    real_write_handler_enabled: bool = False,
) -> None:
    self._audit_store = audit_store or ApprovalGateAuditStoreStub()
    # v2 升级:5th gate 显式 launch flag 严判(沿撞坑 #18 风险门控)
    self._real_write_handler_enabled = (
        real_write_handler_enabled
        and os.environ.get(_ENABLE_PATH_4_WRITE_ENV, "").strip().lower()
        in {"1", "true", "yes", "on"}
    )
```

### 6.2 真实 service 调用失败(异常透传 + audit 落档)

```python
def _call_service_approve_outbox(self, target_id: str, audit: AuditContext) -> WriteResult:
    try:
        result = self._outbox_store.update_status(
            outbox_id=int(target_id),
            new_status="APPROVED",
            from_status="PENDING_SEND",
            last_approved_at_ms=now_ms(),
        )
        # 成功 audit 落档
        audit_id = self._record_audit(
            action="approve_outbox", target_id=target_id, audit=audit,
            write_executed=True, affected_id=str(result.id), error=None,
        )
        return WriteResult(success=True, affected_id=str(result.id), audit_id=audit_id, ...)
    except Exception as e:
        # 失败 audit 落档 + 异常透传
        self._record_audit(
            action="approve_outbox", target_id=target_id, audit=audit,
            write_executed=True, affected_id=None, error=f"{type(e).__name__}: {e}",
        )
        raise  # 异常透传(沿 note_confirm_service.py:113-115)
```
