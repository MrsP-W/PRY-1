**根因初步推测**(待 #94 决策后展开):
1. `build_menu_bar_services()` 内 `Database.open()` 进入 `with` 块后通过 `make_sqlalchemy_engine(db)` 构造 SQLAlchemy engine
2. SQLAlchemy pool 内 **lazy** 创建 connection → 首次 `pool._do_get()` 在 `creator()` 内调用 `db.connection`
3. 实际触发时 `db._closed=True` ⇒ RuntimeError
4. 可能原因:**a** Database `__exit__` 路径异常关闭 / **b** pool creator 内 db 实例状态机与 build_menu_bar_services 上下文不匹配 / **c** sqlcipher 多线程 reader 锁问题

**根因候选**(更细致的 D-step 探索):
- **α**:reader 线程持有 db 实例 → `with Database.open() as db` 退出时 join 失败 → `_closed` 提前置位
- **β**:`build_menu_bar_services` 内部使用 `db` 是 **嵌套 with** 模式 → 内层 close 后外层 SQLAlchemy pool 才尝试 lazy connection
- **γ**:测试环境(SQLAlchemy pool 行为与生产有差异)/launchd 内 `Pool.size` + `pool_pre_ping` 行为差异

**4 修复路径候选**(等 user 决策):
