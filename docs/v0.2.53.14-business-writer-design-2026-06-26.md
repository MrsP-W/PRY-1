> **状态**:📐 docs-only 设计骨架 · **承接**:v0.2.53.11 ApprovalGate 契约 + v0.2.53.12 dry-run 按钮联调 · **边界**:**docs-only**,不写实际代码,**不写 DB / 不接真实 writer / 不发 SMTP / 不读 Keychain** · **下一棒候选**:v0.2.53.15 BusinessWriter Protocol + Stub 落地
>
> **关键缺口**:v0.2.53.11 ApprovalGate 契约已通,env + confirm 双门开启时仍返回 `501 write_not_implemented`,因为 `business_writer_implementation` 尚未实化。**本文档定义该缺口的设计骨架**(只设计,不实现)。

## 一、结论

`POST /api/approval-gate/actions` 契约已具备(env + confirm + 4 类动作白名单 + 审计字段),但 handler 内部没有真实 writer 落地 — 即使 env + confirm 齐全仍返回 `501 write_not_implemented`。

**本骨架**定义 5 件事:

1. **`BusinessWriter` Protocol** — 4 类动作的接口签名
2. **`AuditContext` / `WriteResult` dataclass** — 审计上下文 + 写结果
3. **`AnomalyDismissalService`** — 对应 `finance.dismiss_anomaly` 的新增 Service(沿 #65 opt-in 范本)
4. **三道 env gate** — 默认禁写 + 双确认 + writer 启用
5. **失败降级路径** — 单项失败不传播 + 静默降级 Stub

**关键设计原则**(沿撞坑 #65 + 撞坑 #64 + v0.2.53.8 单项失败降级):

- ❌ 默认禁写(env 未设 → 全 4 类动作一律 `403 write_disabled`)
- ❌ 不接真实业务 writer(本棒 + 至少 v0.2.53.15-18 都不接)
- ❌ 不写 DB / 不发 SMTP / 不读 Keychain 明文
- ✅ 协议先行(Protocol + Stub + 错误码),实化靠后续 v0.2.53.15+ 多 commit 推进

## 二、`BusinessWriter` Protocol 设计

### 2.1 4 类动作方法签名

```python
class BusinessWriter(Protocol):
    """写操作 Protocol — 默认全 no-op,只接契约不接真实写入."""

    def dry_run(
        self,
        action: str,
        target_id: str,
        *,
        audit: AuditContext,
    ) -> WriteDecision: ...

    def approve_outbox(
        self,
        target_id: str,
        *,
        audit: AuditContext,
    ) -> WriteResult: ...

    def cancel_outbox(
        self,
        target_id: str,
        *,
        audit: AuditContext,
    ) -> WriteResult: ...

    def confirm_note(
        self,
        target_id: str,
        *,
        audit: AuditContext,
    ) -> WriteResult: ...

    def dismiss_anomaly(
        self,
        target_id: str,
        *,
        audit: AuditContext,
    ) -> WriteResult: ...
```

### 2.2 4 类动作 → 现有 Service 映射(目标态,v0.2.53.17+ 接入)

| action | 目标态调用 | 现有接口 |
|--------|------------|----------|
| `outbox.approve` | `OutboxStore.update_status(outbox_id, APPROVED, from_status=PENDING_SEND, last_approved_at_ms=now_ms)` | ✅ 已存在(`db/outbox.py:341-461`)|
| `outbox.cancel` | `OutboxStore.update_status(outbox_id, CANCELLED, from_status={PENDING_SEND,APPROVED})` | ✅ 已存在(同上)|
| `notes.confirm` | `NoteConfirmServiceImpl.confirm_note(apple_note_id)` | ✅ 已存在(`menu_bar/note_confirm_service.py:188-206`)|
| `finance.dismiss_anomaly` | **`AnomalyDismissalService.dismiss(anomaly_id, reason=audit.reason)`** | ❌ **不存在** — 本骨架定义新增 |

**关键约束**(沿 D5.6.3 P1-1):

- `dismiss_anomaly` **不收容异常**(同上原则,新 Service 同款)

## 三、`AuditContext` dataclass 设计

```python
@dataclass(frozen=True, slots=True)
class AuditContext:
    """审计上下文 — 每一次写操作必须携带."""
    actor: str                    # 限 80 字符,默认 "local_dashboard"
    reason: str                   # 限 240 字符,默认 ""
    source: str = "dashboard"     # 来源标识
    timestamp_ms: int | None = None  # 可选,不传则 now_ms()

    MAX_ACTOR_LEN: ClassVar[int] = 80
    MAX_REASON_LEN: ClassVar[int] = 240
```

- `source` 默认 `"dashboard"`
- 严判方式同 v0.2.53.11(超长 → `ValueError`)

## 四、`WriteResult` + `WriteDecision` dataclass 设计

### 4.1 `WriteResult`(实际写入返回)

```python
@dataclass(frozen=True, slots=True)
class WriteResult:
    """写操作实际结果 — 真实写入后由 Impl 返回."""
    success: bool
    affected_id: str | None       # 写入后 ID(str,与契约对齐)
    error: str | None             # 错误码,如 "outbox_illegal_transition"
    reason: str | None            # 人类可读原因
    audit_id: str | None = None   # 审计日志 ID(可选)
    write_executed: bool = True   # 必为 True(成功路径)/ False(失败路径)
```

### 4.2 `WriteDecision`(dry-run 返回,沿 v0.2.53.11 `_decision()`)

```python
@dataclass(frozen=True, slots=True)
class WriteDecision:
    """dry-run 决策 — 不实际写入,只校验 + 预览."""
    action: str
    target_id: str
    write_enabled: bool           # env + confirm 双门是否齐全
    would_allow: bool             # writer 实现是否就绪
    write_executed: Literal[False]  # 恒为 False
    dry_run: bool
    action_contract: ActionContract | None
    audit: AuditContext
    error: str | None             # write_disabled | confirmation_required | write_not_implemented | ...
    reason: str | None
    required: tuple[str, ...]     # 开启所需条件列表
```

**与 v0.2.53.11 响应字段一一对应**(`approval_gate.py:188-233 _decision()`),**改动为 dataclass 化 + 类型严格化**。

### 5.3 存储设计(target_id 编码)

**Anomaly ID 格式**(沿 v0.2.53.11 dry-run 按钮设计):

```
{date}|{counterparty}|{amount}
例:2026-06-26|星巴克|38.50
```

- `date`:YYYY-MM-DD(异常发生日期)
- `counterparty`:商家名(str)
- `amount`:金额(str,保留 2 位小数)

**Dismissal 存储表**(沿撞坑 #65 opt-in 范本 · v0.2.53.16 实施):

```sql
CREATE TABLE anomaly_dismissals (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    anomaly_id TEXT NOT NULL,           -- date|counterparty|amount
    reason TEXT NOT NULL DEFAULT '',
    actor TEXT NOT NULL DEFAULT 'local_dashboard',
    dismissed_at_ms INTEGER NOT NULL,
    UNIQUE(anomaly_id)                  -- 同 ID 只 dismiss 1 次
);
CREATE INDEX idx_dismissed_at ON anomaly_dismissals(dismissed_at_ms DESC);
```

**注意**:本表**不引入新依赖**,纯 SQLAlchemy + alembic 迁移(沿 v0.2.1 #4 NoteStore 状态机化范本)。

## 七、失败降级路径(沿撞坑 #65 + v0.2.53.8)

### 7.1 单项失败不传播(沿 v0.2.53.8 单项失败降级范本)

```
approve_outbox(target_id) 失败
  → WriteResult(success=False, error="outbox_illegal_transition", ...)
  → 不影响其他 action 的 dry-run 联调
```

### 7.2 静默降级 Stub(沿撞坑 #65 opt-in 4 阶段)

```
DashboardContext.default() 加载 BusinessWriter 时:
  - BUSINESS_WRITER_ENABLED=1 + writer 构造失败 → Stub
  - 任何 import 失败 / DB 不可用 / 异常 → Stub
  - 静默,API 层不崩,响应维持 write_executed=false
```

### 7.3 异常收窄(沿 `note_confirm_service.py:113-115`)

`confirm_note` 和 `dismiss_anomaly` 是用户主动操作,**不收容异常**:

```python
def confirm_note(self, apple_note_id: str) -> None:
    """用户主动操作,异常必须透传(沿 note_confirm_service.py:113-115)."""
    try:
        self._note_store.mark_archived(apple_note_id)
    except Exception:
        raise  # 不收容
```

`approve_outbox` / `cancel_outbox` 同原则。

### 7.4 dry-run 不传播异常

```python
def dry_run(self, action, target_id, *, audit) -> WriteDecision:
    try:
        return self._compute_decision(action, target_id, audit)
    except Exception as e:
        return WriteDecision(
            ...,
            error="internal_error",
            reason=f"dry_run 异常:{type(e).__name__}",
            write_executed=False,
        )
```
