## 🚨 真实 SMTP 发送 spike 阻塞点

### 阻塞点 1:Gmail OAuth 实际 flow 未跑

- v0.2.2 #5 commit 5 依赖加锁(`6a0549e`)只加 `msal>=1.24` + `google-auth>=2.23` + `google-auth-oauthlib>=1.0`,**未跑真实 OAuth flow**
- 真实 Gmail OAuth flow 需:Microsoft/Google 账号 + App 注册 + 用户授权 + refresh_token 持久化
- **解封条件**:用户手动授权 + 提供 OAuth client_id / client_secret

### 阻塞点 2:`--smtp-provider {qq}` 白名单

- 当前 spike_send_100.py `--smtp-provider` 仅接受 `qq`
- outlook/gmail 凭证写入脚本未实现(B 类延后)
- **解封条件**:沿 v0.2.2 #8 收口报告 "D5 阶段只验证 QQ SMTP 真实链路,outlook/gmail 凭证写入脚本未实现"

### 阻塞点 3:OutboxDispatcher 与 SMTPProviderFactory 集成

- v0.2.2 #8 SMTPProviderFactory 工厂模式实化后,OutboxDispatcher **未集成**
- 当前 spike_send_100.py 沿 `smtp_provider` 参数调用,但 OutboxDispatcher.run_once 仍用 `smtp_host` / `smtp_port` 直连(无 factory 调度)
- **解封条件**:OutboxDispatcher.run_once 加 `provider` 参数 + SMTPProviderFactory.create() 调度
