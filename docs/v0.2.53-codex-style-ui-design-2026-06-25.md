# v0.2.53 Codex 风格 UI 设计稿(2026-06-25 · docs-only)

> 范围:只做 UI/UX 设计沉淀,不改代码、不新增依赖、不真启动菜单栏、不真发邮件、不写凭据。

## 1. 结论

推荐把"我的AI员工"设计成 **Codex 式本地工作台**:

- 左侧是能力导航:邮件 / 笔记 / 财务 / 日程 / 报告 / 系统。
- 中间是任务线程:像 Codex thread 一样展示"用户指令 → 计划 → 执行 → 证据 → 结果"。
- 右侧是上下文检查器:显示选中任务的草稿、账单、笔记、风险门控、证据文件。
- 底部是可折叠执行日志:只在需要排错时展开,默认不打扰。

核心不是做一个花哨 Dashboard,而是做一个 **可确认、可追踪、可恢复的个人 AI 操作台**。

## 2. 参考 Codex 的设计原则

| Codex 启发 | 本项目转译 |
|------------|------------|
| Thread-first | 每个任务都是一个可追踪线程,保留计划、执行、证据、结果 |
| Workspace-aware | UI 始终展示本地数据范围、质量门、凭据状态、隐私边界 |
| Plan + Execute | 高风险动作先列计划,用户确认后执行 |
| Artifact side panel | 草稿邮件、月报、账单候选、笔记结构化结果在右侧预览 |
| Tool trace 可折叠 | 执行日志默认折叠,需要时展开看命令、退出码、文件证据 |
| Approval gate | 发邮件、写 Keychain、真实网络、导入真账单、launchd 都必须显式确认 |

## 5. 核心页面设计

### 5.1 今日页

目标:打开后 10 秒知道"今天要我点什么"。

模块:

- 顶部状态条:本地运行、数据库、LLM、SMTP、Notes、账单导入、质量门。
- 今日待办卡:
  - 邮件草稿待审批
  - Notes 候选待确认
  - 财务异常待查看
  - 日程冲突待处理
- 下一步推荐:
  - 主按钮:"处理最高风险项"
  - 次按钮:"查看全部证据"
  - 危险按钮必须有二次确认:"真实发送 1 封测试邮件"

### 5.2 邮件页

参考 Codex 的"diff + approval"体验,把邮件发送做成可审阅操作。

布局:

- 左:邮件队列和分类标签(URGENT / TODO / FYI / SPAM / PERSONAL)。
- 中:当前邮件线程、AI 草稿、审计意见。
- 右:发送门控:
  - 收件人白名单
  - 黑名单二次检查
  - SMTP provider
  - Keychain 状态(只显示 present/missing,不显示明文)
  - `SMTP_REAL_NETWORK` 状态

按钮:

- 保存草稿
- 退回重写
- 1-click 审批
- 真实发送(默认禁用,需满足 5 重门控)

### 5.3 笔记页

目标:把 ⌥⌘N 捕获后的内容变成可确认知识。

模块:

- 捕获流:最近剪贴板/Notes 同步记录。
- 结构化预览:标题、分类、标签、摘要、是否私密。
- 候选去重:可能重复的旧笔记。
- 一键确认:确认 / 合并 / 私密跳过 / 归档。

### 5.4 财务页

目标:从"账单导入"升级到"财务异常工作台"。

模块:

- 本月概览:收入、支出、净额、异常数。
- 交易列表:按来源、分类、金额、是否候选过滤。
- 异常卡片:商家、金额、原因、历史均值、建议动作。
- 候选 review:确认 / 合并 / 标记非重复。

### 5.5 报告页

目标:所有阶段报告可检索、可复盘。

模块:

- 月度复盘
- release tag readiness
- 撞坑清单
- 质量门趋势
- 最近 commit 证据

### 5.6 系统页

目标:像 Codex 一样透明展示"我现在能不能安全执行"。

模块:

- Git 状态:branch、HEAD、dirty/clean、tag。
- 质量门:pytest、mypy、ruff、format、lint、alembic、build。
- Provider 状态:LLM、IMAP、SMTP、Notes、Keychain。
- 安全门控:
  - 真实 SMTP
  - 真实账单导入
  - launchd/kickstart
  - Keychain 写入

## 6. 关键组件

| 组件 | 用途 |
|------|------|
| StatusPill | 顶部状态胶囊,如"本地运行""SMTP 未解锁" |
| TaskThread | 中心任务线程,展示计划/执行/结果 |
| EvidenceChip | 文件、commit、命令结果的可点击证据 |
| ApprovalGate | 高风险动作确认卡 |
| DraftPreview | 邮件草稿预览 |
| RiskMatrix | tag / SMTP / 账单导入等 7/8 readiness |
| AgentCard | @管家 / @审计员 / @内容编辑员 当前动作 |
| ToolLog | 折叠执行日志 |
| RecoveryCard | 撞坑恢复步骤 |
