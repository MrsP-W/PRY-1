## 3. 实现要点

### 3.1 `DashboardContext.default()` 升级

```python
@classmethod
def default(cls) -> DashboardContext:
    ctx = cls()
    if _is_real_db_enabled():
        real_service = _try_build_real_outbox_drafts()
        if real_service is not None:
            ctx = ctx.with_outbox_drafts(real_service)
    return ctx
```

### 3.2 构造器链(`_try_build_real_outbox_drafts`)

```python
Database.open()  # → 读 Keychain DB 密码 + 打开 SQLCipher
  ↓
make_sqlalchemy_engine(db)  # → SQLAlchemy Engine
  ↓
sessionmaker(bind=engine, expire_on_commit=False)  # → session_factory
  ↓
OutboxStore(session_factory)  # → ORM Store(只读 by_status)
  ↓
OutboxDraftServiceImpl(store)  # → 含 safe_list/safe_count 兜底
```

**任一步失败 → 返回 None → ctx 降级 Stub**(沿 v0.2.53.4 safe_list/safe_count 兜底范本)。

### 3.3 `with_outbox_drafts` 不可变更新范本

```python
def with_outbox_drafts(self, service: OutboxDraftService) -> DashboardContext:
    """返回替换 outbox_draft_service 的新 ctx(不可变更新)."""
    return DashboardContext(
        expense_service=self.expense_service,
        ...
        outbox_draft_service=service,
        ...
    )
```

沿 v0.2.52.3 撞坑 #64 公共 API 迁移范本 — `DashboardContext` 公开 `with_outbox_drafts` 替换服务,而非读私有字段。

## 5. 撞坑 #65(本轮新增)— Dashboard opt-in 真实数据 4 阶段范本

**范本**:任何"Dashboard / UI 默认全 Stub,opt-in 真实数据"场景:
1. **env 门控**:仅 truthy 字面量(`1` / `true` / `yes` / `on`)触发,防误触
2. **默认路径零 I/O**:`DashboardContext.default()` 不调用 DB / Keychain / subprocess
3. **失败静默降级**:任何异常(ImportError / FileNotFoundError / PermissionError / 错密码)→ `None` → 降级 Stub
4. **不可变更新**:用 `with_outbox_drafts(service)` 替换服务,沿 #64 公共 API 范本

### 5.1 范本适用场景

- ✅ Dashboard 默认只读,真实数据需 opt-in
- ✅ 任何"开/关"敏感操作的默认行为(读 DB / 写 DB / 调外部 API)
- ✅ 用户授权后自动启用,失败仍可用(降级)
- ❌ 不适用:必须真实数据才能工作的场景(此时 opt-in 反而是负担)
