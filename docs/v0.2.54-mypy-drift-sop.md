## 2. 撞坑 #69 SOP — mypy 严格模式升级 / type: ignore 注释清理

### 2.1 适用场景

- mypy 严格模式升级(从 `--strict` 到更严版本)
- 批量清理冗余 / unused type: ignore 注释
- 撞坑 #69 复发的预防

### 2.2 SOP 步骤

| 步骤 | 动作 | 工具 |
|------|------|------|
| **1. 备份 type: ignore 注释清单** | `git grep -n "# type: ignore" src tests > type-ignore-backup-$(date +%Y-%m-%d).txt` | git + grep |
| **2. 分类** | 按 [arg-type] / [union-attr] / [misc] / [return-value] / [assignment] 等 11 类分类(沿 v0.2.53.41 §1.1)| 手工 + 脚本 |
| **3. 验证每个 type: ignore 的必要性** | 每个 type: ignore 注释对应一个 mypy error 编号(沿 v0.2.53.40 撞坑 #69)| mypy --strict |
| **4. 删废弃 type: ignore** | 只删已被其他修复(冗余 + unused)的 type: ignore | 沿 v0.2.53.40 + v0.2.53.41 修复记录 |
| **5. 改写必要的 type: ignore** | 改用 type stub / Protocol / cast 等更精确表达(沿 v0.2.23 cast 范本)| mypy 严格模式 |
| **6. 验证** | `mypy --strict src tests` 必须 0 errors,且 type: ignore 数量下降 | mypy |
| **7. docs-only 收口** | 写入 `docs/v0.2.X.Y-type-ignore-cleanup.md` + `quality_snapshot.py` L20 claim 同步 | docs-only |

### 2.3 撞坑 #69 预防 checklist

- [ ] type: ignore 注释备份清单
- [ ] 每个 type: ignore 对应错误编号
- [ ] 批量脚本**不直接删除** type: ignore(改用 mypy 输出验证)
- [ ] 修改后 mypy --strict 0 errors 验证
- [ ] docs-only 收口 + `quality_snapshot.py` 同步
- [ ] 沿用本项目负向类型测试范本(`tests/core/test_oauth2.py` 等)

### 2.4 沿用边界

- ❌ **不批量删除** type: ignore(撞坑 #69 根因)
- ❌ **不跳过**负向类型测试(撞坑 #69 误删原因)
- ❌ **不前进** mypy 严格模式到 mypy 0.9xx(等社区稳定)

## 3. 撞坑 #70 SOP — 中文注释 + type:ignore 同行问题

### 3.1 适用场景

- 新增任何带中文注释 + type: ignore 的代码行
- 撞坑 #70 复发的预防
- 任何 `# type: ignore` 注释的代码审查

### 3.2 SOP 步骤

| 步骤 | 动作 | 工具 |
|------|------|------|
| **1. 中文注释独立行** | 中文注释必须**独立成行**,不与 type: ignore 同行 | 代码审查 |
| **2. type: ignore 行尾规则** | type: ignore 必须位于**行尾**或**单独行** | 代码审查 |
| **3. 验证** | `mypy --strict src tests` 必须识别所有 type: ignore | mypy |
| **4. docs-only 收口** | 写入 type: ignore 注释规范到 `CONTRIBUTING.md` 或 `docs/v0.2.X.Y-type-ignore-rule.md` | docs-only |

### 3.3 撞坑 #70 预防 checklist(每次新增 type: ignore 时)

- [ ] 中文注释独立行(**不与 type: ignore 同行**)
- [ ] type: ignore 位于**行尾**或**单独行**
- [ ] mypy --strict 验证识别
- [ ] v0.2.53.41 hotfix 8 files 检查(沿 `0d21b50` 修复)

### 3.4 中文注释 + type:ignore 反例 vs 正例

**反例**(撞坑 #70 触发):
```python
# 这是负向测试,故意传错类型 # type: ignore[arg-type]
bad_value: int = "string"
```

**正例 1**(中文注释独立行 + type:ignore 行尾):
```python
# 这是负向测试,故意传错类型
bad_value: int = "string"  # type: ignore[arg-type]
```

**正例 2**(type:ignore 单独行):
```python
# 这是负向测试,故意传错类型
# type: ignore[arg-type]
bad_value: int = "string"
```

### 3.5 沿用边界

- ❌ **不在中文注释同行加 type: ignore**(撞坑 #70 根因)
- ✅ 中文注释独立行 + type:ignore 行尾/单独行
- ✅ 任何 PR 含中文注释 + type:ignore → 必须经此 SOP 审查
