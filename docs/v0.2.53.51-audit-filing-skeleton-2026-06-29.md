## 2. 设计决策

### 2.1 撞坑 #18 风险门控应用

- 默认 `BusinessWriterImpl._real_write_handler_enabled=False` 锁定(沿 v0.2.53.49)
- 真实写入路径留 8/1 后 + 用户明确授权 + handler 路径 4 启用
- **audit 落档仅在写保护锁开 + 真实 service 调用(成功/失败)后发生**
- 4 类**不落档**场景:
  1. **dry-run 决策**:不调 `audit_store.record()`
  2. **写保护锁 raise**:`_real_write_handler_enabled=False` 时不调
  3. **invalid_target_id**:参数校验失败(用户输入问题),不污染日志
  4. **依赖未注入**:`_check_dep` 抛 NotImplementedError,不调 service,不调 audit

### 2.2 撞坑 #64 公共 API 一致性

- `audit_id` 字符串格式 `"audit:{id}"` 与 `anomaly_dismissals` `"dismissal:{id}"` 对齐
- `ApprovalGateAuditStoreStub` 暴露 `get_default_stub()` 工厂(沿 AnomalyDismissalServiceStub 范本)
- `InMemoryApprovalGateAuditStore` 字段 `_enabled`(下划线前缀)避开与 `Protocol.is_enabled()` 方法同名冲突(沿 v0.2.53.7 Stub 范本)

### 2.3 撞坑 #65 opt-in 4 阶段范本

- `audit_store: ApprovalGateAuditStore | None = None` 默认参数
- `None` → `ApprovalGateAuditStoreStub()`(等效 `is_enabled=False`)
- 实际生产路径:`DashboardContext.default()` 在 `DASHBOARD_REAL_DB=1` 时尝试构造 `InMemoryApprovalGateAuditStore`(沿 v0.2.53.27 范本)
- 单项失败静默降级 Stub(沿 v0.2.53.7 范本)

### 2.4 D5.6.3 P1-1 审批凭据必传规则

- `approve_outbox` 必须传 `last_approved_at_ms=now_ms`(int)
- `cancel_outbox / confirm_note / dismiss_anomaly` 必传 `last_approved_at_ms=None`

### 2.5 v0.2.53.11 write_executed 不变式(沿用)

- dry-run 上下文恒 `False`
- 真实写入上下文恒 `True`(本棒真实调 service 时)
- `WriteResult.audit_id` 字段从 `None` 升级为真实 `audit_id` 字符串(`"audit:1"`、`"audit:2"`、...)
