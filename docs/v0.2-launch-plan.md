## v0.2.53 Codex 风格 UI 工作台计划(2026-06-25 新增)

**结论**:把"我的AI员工"从菜单栏能力集合升级为 **Codex 式本地工作台**,先做 P0 静态原型验证信息架构,再决定是否进入本地 Web Dashboard。

### 目标

- 让用户 10 秒内知道"今天要审批什么 / 哪些风险被拦住 / 证据在哪里"。
- 用任务线程展示"指令 → 计划 → 执行 → 证据 → 结果",避免黑箱自动化。
- 所有高风险动作继续走显式确认:真实 SMTP、Keychain 写入、真实账单导入、launchd/kickstart、tag 创建。
- 继续保持本地优先、隐私优先,UI 不绕过现有 Outbox / Notes / Finance 状态机。

| 阶段 | 产物 | 范围 | 质量门 |

| **P0 静态原型** | `docs/ui/codex-style-dashboard.html` + `docs/ui/codex-style-dashboard.md` | 今日页 / 邮件审批页 / 系统健康页,只用假数据 | `make lint` |
