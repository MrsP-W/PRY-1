**注释说"防止我以为是 1 封但实际 10"** — 这是撞坑 #78 暴露的设计意图(严判 count 必须 == 1,防误跑 10 封)。

## 3. 撞坑 #78 暴露的问题

- **10 封 spike 跑通了**(v0.2.55.5 sent=10 tech_fail=0)
- **100 封 spike 不可行**:`--count 必传 1` 严判 → 必须跑 100 批 × 1 封 = 100 分钟
- **QQ SMTP 频次/反垃圾风险**:100 封需评估撞坑 #78 R1(频次限制)+ R3(反垃圾识别)
- **90 封 spike 真实邮件是 outward-facing 风险**:撞坑 #18 风险门控 = 真实邮件发出去 = 公开

**撞坑 #78 决策**:100 封 spike 拆 2 棒(10 封本棒 + 90 封下一棒),但 90 封 spike 仍需 100 批 × 1 封 = 90 分钟,总时长 100 分钟。

**D5.6.3 解决思路**:放宽 `--count 1-10 可配置`,允许 1 批跑 1-10 封,90 封 = 9 批 × 10 封 × 60-120s ≈ 15-20 分钟。

## 4. 放宽方案

### 4.1 核心修改(L382-384 → L382-394)

```python
# 4. D5.6.3 spike 严判放宽:--real 模式 --count 范围 1-10(沿用 _REAL_MODE_MAX_COUNT)
if count < 1 or count > _REAL_MODE_MAX_COUNT:
    raise ValueError(
        f"D5.6.3 防误发:--real 模式 --count 必传 1-{_REAL_MODE_MAX_COUNT},实际 {count!r}"
    )

# 4b. D5.6.3 多重防御:--count > 1 必传额外 confirm(防"我以为是 1 封但实际 10")
if count > 1 and multi_confirm != _MULTI_CONFIRM_PHRASE:
    raise ValueError(
        f"D5.6.3 防误发:--real 模式 --count > 1 时必传 {_MULTI_CONFIRM_PHRASE!r},"
        f"实际 {multi_confirm!r}(--count=1 时不需)"
    )
```

### 4.2 新增常量(L77 附近)

```python
_MULTI_CONFIRM_PHRASE = "yes-i-understand-batch-send"  # count > 1 时必传
# 沿 _REAL_MODE_MAX_COUNT=10 / _REAL_MODE_MAX_RECIPIENTS=1 / _CONFIRM_PHRASE 既有
```

### 4.3 CLI 参数新增(L34 附近)

```python
"--multi-confirm": {
    default="",
    help="D5.6.3 防误发:--real 模式 --count > 1 时必传 yes-i-understand-batch-send"
}
```

### 4.4 严判顺序(沿 L367-393 既有顺序)

```text
0. --real 模式必传 + SMTP_REAL_NETWORK=1
1. --max-recipients 必传且 == 1
2. --confirm 必传且 == _CONFIRM_PHRASE(向后兼容 count=1)
3. **--count 范围 1-_REAL_MODE_MAX_COUNT**(本次改)
4. **--count > 1 时 --multi-confirm 必传 _MULTI_CONFIRM_PHRASE**(本次新增)
5. smtp_host / smtp_username 不能是占位
6. Keychain 真读 16 chars 命中
```

## 5. 多重防御设计(撞坑 #18 风险门控)

| # | 门控 | 严判 | 撞坑防御 |
|---|------|------|---------|
| 1 | `SMTP_REAL_NETWORK=1` | 默认拒绝,显式 1 解锁 | #18 防误发 |
| 2 | `--max-recipients 1` | `--real 模式强制 1 收件人` | #18 防 spam |
| 3 | `--confirm "yes-i-understand-this-sends-real-email"` | 精确字符串匹配(向后兼容) | #18 防误发 |
| 4 | **`--count 1-10`** | **本次放宽**,沿 `_REAL_MODE_MAX_COUNT` | #78 防超量 |
| 5 | **`--multi-confirm "yes-i-understand-batch-send"`** | **`count > 1` 必传**,`count=1` 不需 | #78 防误以为是 1 |
| 6 | `--recipient` 白名单 | 自发自收 477753009@qq.com | #18 防 spam |
| 7 | `--smtp-host` / `--smtp-username` 非占位 | `.test.local` / `spike@qq.com` 拒绝 | D5.6.2 占位防误发 |
| 8 | Keychain round-trip | 16 chars 命中,不打印 value | #16/#17 防泄露 |
| 9 | **`--max-batch-iterations`** | **新增上限 100**,防止脚本反复循环 | #78 防 runaway |

**多重防御总和**:9 重门控,撞坑 #18 风险门控 = "发出去 = 公开",必须每重都通过。
