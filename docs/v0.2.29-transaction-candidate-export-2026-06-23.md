## 1. 目标

当前没有真实微信/支付宝 CSV 路径,不能启动 W3 真账单 `--max-rows 1` 小样本导入。

因此执行备选动作:把现有 `needs_confirm=1` 跨源候选变成可人工 review 的稳定导出产物,避免候选只停留在数据库或 spike 报告里。

## 2. 改动

### 2.1 Store 只读热路径

新增:

```python
TransactionStore.list_by_needs_confirm(limit=100, source_filter=None)
```

行为:

- 只查 `needs_confirm=1`
- 可选按 `source_filter` 过滤新交易来源
- 按 `imported_at_ms DESC` 排序
- 严判 `limit` 范围 `[1,10000]`
- 不修改 `status / needs_confirm / candidate_match_id`
- 导出的 `reports/transaction-candidates*.csv/jsonl` 视作本地敏感 review 产物,已加入 `.gitignore`

### 2.2 导出脚本

新增:

```bash
uv run python scripts/export_transaction_candidates.py --format jsonl
uv run python scripts/export_transaction_candidates.py --format csv --output-path reports/transaction-candidates.csv
```

安全约束:

- `--limit` 预检范围 `[1,10000]`,错误时 exit 1,不打开 DB
- `--source` 会先 strip 并拒绝空字符串,再交给 Store 层做 source pattern 严判
- 导出文件可能包含真实商户/金额/交易号,只做本机 review,不提交 Git

输出字段覆盖:

- 新交易:`tx_id/source/external_transaction_id/date/amount/counterparty/category/payment_method/status/fingerprint`
- 候选交易:`candidate_match_id/source/external_transaction_id/date/amount/counterparty/category/payment_method`
- 异常兜底:`candidate_missing`
