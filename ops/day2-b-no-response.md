## §2 可能根因(3 候选 · 未实测验证)

### 2.1 候选 1:macOS TCC 辅助功能未授权(最可能)

- `⌥⌘N` 全局快捷键需要 **辅助功能** 权限(系统设置 → 隐私与安全性 → 辅助功能)
- 菜单项点击本身不需要 TCC,但 rumps 在某些 macOS Sequoia/Tahoe 上需 Terminal/iTerm 加 **自动化** 权限
- **修复**:Day 2 用户补授权后重试

### 2.2 候选 2:macOS 焦点问题(Terminal 抢焦点)

- 用户启动菜单栏后,Terminal 仍是焦点应用
- macOS 在 Terminal 焦点下,菜单栏点击事件可能延迟或被吞
- **修复**:点击 macOS 桌面空白处,让菜单栏收焦点后重试

### 2.3 候选 3:rumps 0.4.0 + Python 3.12 兼容性

- 项目用 `rumps 0.4.0`(撞坑 #11 沿用 · 6/16 范本)
- rumps 在某些 macOS 版本上 callback 不触发
- **修复**:Week 2 可考虑升级 rumps 或迁移到 pystray
