## 2. 关键决策详解

### 2.1 依赖策略：PEP 621 + uv

#### 原状（D1）

```toml
# pyproject.toml
[tool.poetry.dependencies]
python = "^3.11"  # 实际跑 3.14.4
imapclient = "^3.0.1"
...

[tool.poetry.group.dev.dependencies]
pytest = "^8.3.4"
...
```

```gitignore
# .gitignore
poetry.lock  # ⚠️ 忽略锁文件！
```

```makefile
# Makefile
uv pip install -e ".[dev]"  # ⚠️ uv 不识别 poetry group
```

**问题**：
- `.gitignore` 忽略 `poetry.lock` → 锁版本失败
- `uv pip install -e ".[dev]"` 不识别 `[tool.poetry.group.dev]` → dev 依赖装不上
- pyproject.toml 写 `^3.11` 实际跑 3.14.4 → 解释器不可复现

#### 修正后（D1.1）

```toml
# pyproject.toml
[project]
name = "my-ai-employee"
version = "0.1.0"
requires-python = ">=3.12,<3.13"
dependencies = [
    "imapclient>=3.0.1",
    "sqlcipher3>=0.6.2",  # 替代 pysqlcipher3
    ...
]

[project.optional-dependencies]
dev = [
    "pytest>=8.3.4",
    "watchdog>=6.0.0",  # dev 模式 hot reload
    ...
]

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[tool.hatch.build.targets.wheel]
packages = ["src/my_ai_employee"]
```

```gitignore
# .gitignore
.venv/
# uv.lock 提交（uv 官方推荐）
node_modules/
# package-lock.json 提交
```

```makefile
# Makefile
install:
    uv sync --extra dev && uv pip install --python .venv/bin/python -e .
```

**效果**：
- ✅ `uv sync --extra dev` 装全部依赖（31 个）
- ✅ `uv.lock` 118KB 提交，保证可复现
- ✅ 标准 PEP 621 格式，pip/uv/pdm/poetry 全部识别

### 2.2 Python 3.12 固定

#### 风险验证

D1 报告标红的 "pysqlcipher3 高风险" **今天兑现**：

```bash
$ uv pip install pysqlcipher3
...
ValueError: No distribution was found. Ensure that `setup.py` is not empty
  and that it calls `setup()`.
```

源包 setup.py 是空的（2020 年起没修），Python 3.12 wheel 缺失。

#### 降级方案

试了 3 个替代：

| 包 | 状态 |
|----|------|
| `pysqlcipher3` 1.2.0 | ❌ wheel 缺失，setup.py 空 |
| `pysqlcipher3-binary` | ❌ 同上（共享 setup.py）|
| **`sqlcipher3` 0.6.2**（coleifer fork）| ✅ PyPI 有 Python 3.12 wheel，867ms 装好 |

**为什么 `sqlcipher3` 靠谱**：
- 作者：Charles Leifer（pysqlite3 / peewee 作者，Python 圈大佬）
- 维护：活跃（最新 2026-05-01 commit）
- API：与 pysqlcipher3 兼容（同样的 `PRAGMA key` 流程）
- DB 文件：可互迁

#### 修正后

```toml
# pyproject.toml
requires-python = ">=3.12,<3.13"
dependencies = [
    "sqlcipher3>=0.6.2",  # 替代 pysqlcipher3
    ...
]
```

```bash
# .python-version
3.12
```

```bash
# .venv/bin/python --version
Python 3.12.13
```

### 2.3 Lint 工具固定

#### 原状

```makefile
lint:
    @command -v markdownlint-cli2 >/dev/null 2>&1 && \
        markdownlint-cli2 "**/*.md" || \
        npx --yes markdownlint-cli2 "**/*.md"
```

**问题**：
- 优先全局工具（版本不定）
- 降级 npx（受网络/版本影响）
- 静默 fallthrough，**用户不知道走哪条路**

#### 修正后

`package.json`：

```json
{
  "name": "my-ai-employee-lint",
  "private": true,
  "devDependencies": {
    "markdownlint-cli2": "0.22.1"
  }
}
```

`Makefile`：

```makefile
lint:
    @if [ -x node_modules/.bin/markdownlint-cli2 ]; then \
        node_modules/.bin/markdownlint-cli2 "**/*.md" || exit 1; \
    elif command -v markdownlint-cli2 >/dev/null 2>&1; then \
        markdownlint-cli2 "**/*.md" || exit 1; \
    else \
        echo "❌ markdownlint-cli2 未安装"; \
        echo "  请先跑: make install-npm"; \
        exit 1; \
    fi
```

`scripts/pre-commit`：

```bash
# 缺失时显式失败（不再静默跳过）
if [ -x node_modules/.bin/markdownlint-cli2 ]; then
    ...
elif command -v markdownlint-cli2 >/dev/null 2>&1; then
    ...
else
    echo "❌ markdownlint-cli2 未安装 — 拒绝提交" >&2
    echo "" >&2
    echo "  安装方式（任选其一）：" >&2
    echo "    1. 项目级（推荐）：make install-npm" >&2
    echo "    2. 全局：         npm install -g markdownlint-cli2@0.22.1" >&2
    exit 1
fi
```

### 2.4 包名重构 `src/` → `src/my_ai_employee/`

#### 原状

```toml
[tool.poetry]
packages = [{ include = "src" }]
```

```python
# src/main.py
from src import __version__
```

```bash
python -m src.main
```

**问题**：
- `from src import ...` 不是标准命名空间
- 测试要 `pythonpath = ["src"]` hack
- 工具链（mypy/ruff）经常误识别

#### 修正后

```toml
[tool.hatch.build.targets.wheel]
packages = ["src/my_ai_employee"]
```

```python
# src/my_ai_employee/main.py
from my_ai_employee import __version__
```

```bash
python -m my_ai_employee.main
```

**重构步骤**（git mv 保留历史）：

```bash
mkdir -p src/my_ai_employee
git mv src/__init__.py src/my_ai_employee/__init__.py
git mv src/connectors src/my_ai_employee/connectors
git mv src/core       src/my_ai_employee/core
git mv src/ai         src/my_ai_employee/ai
git mv src/agents     src/my_ai_employee/agents
git mv src/menu_bar   src/my_ai_employee/menu_bar
```

### 2.5 测试覆盖率提升（0% → 61.9%）

#### 原状

```python
# tests/test_main.py
def _run_main(*args):
    return subprocess.run(
        [sys.executable, "-m", "src.main", *args],
        cwd=PROJECT_ROOT, capture_output=True, text=True, timeout=15,
    )
```

**6 测试全 subprocess** → main.py 覆盖率 **0%**

#### 修正后

D1.1 拆出 5 个可单测纯函数：

| 纯函数 | 用途 |
|--------|------|
| `build_parser()` | CLI 解析器（参数注入 `argv`）|
| `_strip_rich_tags(line)` | 降级模式去标签 |
| `render_hello_body(version, now, py_ver, platform, use_rich)` | Hello 面板 body |
| `render_info_body(project_root, data_dir, source_entry, ...)` | Info 面板 body |
| `main(argv=None)` | 主入口（argv 可注入）|

**18 个测试**：

| 类型 | 数量 | 覆盖 |
|------|------|------|
| 纯函数单测 | 9 | build_parser × 3、_strip_rich_tags × 2、render_*_body × 4 |
| Subprocess 冒烟 | 2 | CLI hello + --version |
| 包元数据 | 2 | `__version__` / `__author__` |
| 参数化边界 | 5 | `_strip_rich_tags` 的 5 个标签组合 |
| **总计** | **18** | **覆盖率 61.9%** |

**剩余未覆盖**（35%）：`_print_panel` / `print_hello` / `print_info` / `run_interactive` — 都是副作用（调 rich console / print），D2+ 接入菜单栏后用 mock 进一步提升。

## 6. 关键发现 & 经验

### 6.1 pysqlcipher3 死透了

D1 标红的风险**今天兑现**：pysqlcipher3 1.2.0 在 Python 3.12 上**根本装不上**。

**教训**：以后选 SQLite 加密库，**首选 `sqlcipher3`（coleifer fork）**，避开老 pysqlcipher3 链。

### 6.2 应急版范本 = 真实救命

D1 装不上 pysqlcipher3 时，没有 panic —— 按"试替代"路径找到 sqlcipher3 一次过。

**沉淀**：以后遇到装不上的老库，**先搜 GitHub "active fork"**，不要死磕。

### 6.3 包名重构的 0 成本窗口

D1 文件少（23 个），改包名 5 分钟做完；D2 之后有 50+ 文件时改 = 灾难。

**沉淀**：包名/包结构决策**永远在脚手架阶段做**，不要拖。

### 6.4 应急版范本在 pre-commit 的应用

旧 pre-commit 在工具缺失时静默跳过（`exit 0`）—— **这等于"提交前自动 MD lint"是虚的**。

修正：缺失时**显式失败** + 给出明确安装命令。

**沉淀**：所有"自动检查"都应该是 hard fail，**silent skip = silent corruption**。
