## 🔬 9 场景详细验证

### S1 新邮件到达 → IMAP 轮询 → 分类

- **链路**:D2 IMAP 拉取 → D4.6 5 类分类(URGENT/TODO/FYI/SPAM/PERSONAL)
- **范本**:InMemory faker 10 封邮件 + D4.6 LLM mock 返回固定 JSON
- **关键断言**:10 封全 5 类分布 + 分类延迟 < 5000ms
- **沿 D5.6.5 真实 1 封范本**:默认 InMemory,真实 IMAP 需 `IMAP_REAL_NETWORK=1` env

### S2 1-click 草稿生成 → drafter < 10s

- **链路**:D4.6 分类 → D4.7 drafter 5+1 类 SYSTEM prompt
- **范本**:InMemory 1 封 URGENT 邮件 → drafter 5s 内生成草稿
- **关键断言**:drafter latency < 10000ms + 5+1 类 prompt 注入正确
- **沿 D4.7.2 v1.0.8 契约**:`DraftResult` 强一致字段 + 抗注入 `UNTRUSTED_DATA` 包裹

### S3 草稿入库 → outbox 表

- **链路**:D4.7 drafter → D4.8 outbox 表 UNIQUE 幂等
- **范本**:InMemory 1 封草稿 → outbox.insert + status=PENDING_SEND
- **关键断言**:outbox 1 行 + status=PENDING_SEND + UNIQUE 幂等
- **沿 D4.8 v1.0.1 契约**:`OutboxEntry` 11 字段 + LaneBoard.ACTIVE

### S4 1-click 审批 → 审批状态推进

- **链路**:D4.8 outbox PENDING_SEND → 1-click approve → status=APPROVED
- **范本**:InMemory 1 个 outbox PENDING_SEND → approve → status=APPROVED
- **关键断言**:outbox status 状态机推进 + 1-click 幂等
- **沿 D4.8 v1.0.1 状态机契约**:`PENDING_SEND → APPROVED → SENDING → SENT`

### S5 真实 SMTP 发送(D5.6.5 真实 1 封范本)
