## 2. spike 数据(MOCK 模式,默认)

```
$ uv run python -m scripts.spike_b5_quartz --n 30
quartz spike: mode=MOCK received=30 inserted=29 skipped=1 failed=0
$ echo $?
0
```

**字段解读**:
- `mode=MOCK` — 走 MockRunner(默认,真 macOS 辅助功能授权机器上 `QUARTZ_REAL_TAP=1` 才走 RealQuartzRunner)
- `received=30` — MockRunner 推 30 次 ⌥⌘N hotkey 事件
- `inserted=29` — 前 29 笔模拟 NoteStore.insert 成功
- `skipped=1` — 第 30 笔模拟 NoteDuplicateError(L1 幂等命中)
- `failed=0` — 业务失败 0
- **exit code 0** — 4 退出码契约 0 = 成功

## 3. 4 退出码契约(沿 import_wechat.py + D3.3.3 教训)

| 退出码 | 语义 | 触发场景 |
|--------|------|----------|
| **0** | 成功 | `received == n` 且 `failed == 0` |
| **1** | 解析失败 | `--n` 越界 / Quartz 模块未装 / MockRunner 启动失败 |
| **2** | 业务失败 | `received != n` / `failed > 0`(NoteStore.insert 失败 / 重复扣款) |
| **3** | 技术失败 | Quartz tap 抛 RuntimeError / 异常 / 子进程 spawn 失败 |

**D3.3.3 教训应用**:`except (RuntimeError, OSError)` 收 Quartz tap 异常 + 透传 → exit 3(技术失败),不污染业务失败语义。
