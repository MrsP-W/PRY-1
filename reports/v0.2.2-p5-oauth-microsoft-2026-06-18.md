## 5. 关键设计决策(5 条)

### 5.1 msal 用 `TYPE_CHECKING` + 函数内 import(6/19 commit 2 暂不引入 dep)

**理由**:
- 6/19 commit 2 不写 `pyproject.toml dependencies`,延后到 6/22 commit 5(B 类决策延后)
- 但 launch 文档 §2.1 commit 2 承诺"msal 接入",所以代码必须能调用 `msal.ConfidentialClientApplication`
- 解决:模块顶部 `if TYPE_CHECKING: import msal` + `_build_msal_client` 函数内 `import msal`
- 测试用 `msal_client_factory` 注入 mock,完全不走函数内 import 路径
- 6/22 加 dep 后,生产走真 msal 路径,契约不变

**风险**:6/22 加 dep 失败时,真实生产无法调用 — 6/22 commit 5 必须跑 spike 验证。

### 5.2 显式继承 `OAuth2Provider` Protocol(非 duck typing)

**理由**:
- mypy 能识别类型契约(无需 `# type: ignore[assignment]`)
- ruff 不报 `OAuth2Provider` F401(imported but unused)
- IDE 跳转更友好,新人能直接看到"MicrosoftOAuth2 是 OAuth2Provider 的实化"
- 沿 [[d4.7.3-v1.0.6-p2-3]] 公共 API 自防御范本

### 5.3 `_msal_result_to_token` 内部严判 RFC 6749 §5.2 error 字段

**理由**:
- msal 真实调用可能返回 `{"error": "invalid_grant", "error_description": "..."}` 错误响应
- 必须严判,否则后续访问 `result["access_token"]` 会抛 KeyError(异常类型混乱)
- 错误响应统一抛 `OAuth2TokenExchangeError`(语义对应 RFC 6749 §5.2)
- refresh_token 方法内部用 try/except 转译为 `OAuth2TokenRefreshError`(语义对应 RFC 6749 §6)

### 5.4 `OAuth2TokenExchangeError/RefreshError` 不接 `error`/`error_description` 关键字(沿 Phase 1 范本)

**理由**:
- Phase 1 `oauth2.py:80-86` 定义:`OAuth2TokenExchangeError(OAuth2Error)` / `OAuth2TokenRefreshError(OAuth2Error)` 是 `OAuth2Error(Exception)` 简单子类
- 不同于 `OAuth2AuthorizationError` 有 `__init__` 重载(`error` / `error_description` 属性)
- `Exception` 接受 `*args` 但不接 `**kwargs`,传 kwargs 抛 `TypeError`
- 修复:删除 kwargs,错误信息全部通过 `message` 传递
- 教训:Phase 1 异常类设计应统一(`OAuth2AuthorizationError` 与 `OAuth2TokenExchangeError` 行为不一致,沿 v0.2.3 B 类决策延后)

### 5.5 `msal_client_factory` 注入参数 + 工厂模式(测试 + 生产解耦)

**理由**:
- 6/19 commit 2 测试不需要真实 msal 依赖,`msal_client_factory` 注入 mock 即可
- 6/22 commit 5 加 dep 后,生产用 `MicrosoftOAuth2()` 不传 factory,自动 `import msal` 走真路径
- 范本:D5.6.5 4 重防误发(环境门控 + 强制 1 封 + Keychain round-trip)中的 factory 模式
- 单测无需 monkeypatch 整个模块,工厂注入更显式

## 8. 关键教训(5 条)

- **`OAuth2Error` 子类 kwargs 设计必须一致** — `OAuth2AuthorizationError` 接受 `error` / `error_description`,`OAuth2TokenExchangeError/RefreshError` 不接受 → 首次实现踩 TypeError 坑(B 类决策延后统一异常基类)
- **`TYPE_CHECKING + 函数内 import` 是测试 + 生产解耦范本** — 6/19 测试不依赖真实 msal,6/22 加 dep 后契约不变
- **`isinstance(provider, OAuth2Provider)` 必须显式继承 Protocol** — duck typing 不报 F401 也不报 isinstance 错误,但 IDE 跳转 / mypy 契约会丢失
- **8/8 质量门必跑** — 沿 v0.2.2 范本,单 commit 失败立即回滚(本次踩 4 类问题:OAuth2Error kwargs + F401 + format + mypy invalid type:ignore)
- **`pytest --no-cov` 单文件跑免覆盖误报** — 跑全测试套才走 --cov,单文件 --no-cov(2.4% 误报溯源)
