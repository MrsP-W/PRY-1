## 4. 设计决策(5 关键)

### 4.1 Polling 而非 event-driven(沿 D5 业务调度范本)

**核心思路**:`needs_confirm=1` 触发源太多(sync_notes / IMAP / AppleScript / OutboxDispatcher),**polling 是最稳的**(沿 D5 OutboxDispatcher 范本)。

**复用现有 polling 基础设施**:
- 复用 `_stop_hotkey_poll` Event(主进程退 → 全停)
- 复用 `_BADGE_POLL_SLEEP_GRANULARITY_SECONDS=1.0` 优雅 sleep 范本
- 独立 daemon thread 平行 `_poll_hotkey_queue`,互不干扰

### 4.2 wait granularity 动态调整

```python
wait_granularity = min(_BADGE_POLL_SLEEP_GRANULARITY_SECONDS, interval)
# interval=30s 时 granularity=1.0(生产)
# interval=0.1s 时 granularity=0.1(测试)
```

**为什么不写死 1.0**:
- 短 interval 测试场景(0.1s)用 1.0 granularity → 0.35s 内只触发 1 次(等不到)
- min(1.0, interval) 让测试场景也能正确触发多次 refresh

### 4.3 `__bases__ = (_FakeRumpsApp,)` 强制重绑

**真坑**:`NotesMenuBarApp` 在 import 时已绑定 `_RumpsAppBase` 为基类,**monkeypatch `_RumpsAppBase` 不影响已定义的 class**。`super().__init__` 仍调 `rumps.App.__init__`,产生真 `rumps.Menu`(menu list 形态断言失败)。

**修复**:fixture 内 `app_cls.__bases__ = (_FakeRumpsApp,)` 显式重绑基类。

### 4.4 `_refresh_anomaly_count` 改用 `_update_menu_badge` helper

**Bug 修复**(生产代码,非 #6 新增 bug):
- 原代码 `getattr(item, "title", None)` 在 str 形态下拿到 `str.title` 内置方法(非 None)
- 后续 `item.title = ...` 在 str 上抛 `AttributeError`,静默吞掉
- 改用 `_update_menu_badge` helper(沿 #2 范本),同时支持 list[str] 和 rumps.Menu 2 种形态

### 4.5 `_FakeBadgeConfirmService` 线程安全

`get_pending_confirm_count` 用 `threading.Lock` 保护 `call_count` 计数:
- 防止 polling thread 多次调用 + 测试主线程读 `call_count` race
- `raise_on_count=True` 模拟 service 异常,测 polling 静默容错

## 7. 关键决策沉淀(8 条)

1. **Badge 实时刷新 = polling 30s**(沿 D5 业务调度范本)— 不引入 event-driven
2. **wait granularity 动态** — `min(1.0, interval)` 支持短 interval 测试
3. **首次启动立即刷 1 次** — 避免刚启动 30s 后才显示正确数字
4. **`__bases__ = (_FakeRumpsApp,)` 强制重绑基类** — monkeypatch 失败的根本解法
5. **严判 type/range [0, 3600]** — 入口处拒 bool/str/负数/超大(沿 D4.7.3 v1.0.5)
6. **interval=0 显式禁用** — 测试场景或用户主动选择(沿 D5 范本)
7. **`_FakeBadgeConfirmService` 线程安全** — `threading.Lock` 保护 `call_count`
8. **`_refresh_anomaly_count` 改用 helper** — 修复 str 形态下 `getattr` bound method bug

## 8. 关键教训(6 条)

- **monkeypatch 基类不够** — `class X(B):` 在 import 时已绑定,改 `B` 不影响 `X` 的 `super().__init__`;必须 `X.__bases__ = (NewB,)` 直接改 `__bases__`
- **`str.title` 是内置方法** — `getattr(s, "title", s)` 拿到 `bound method` 而非默认值;查找时必须 `isinstance(item, str)` 短路
- **Polling 间隔粒度匹配 interval** — 写死 1.0 granularity 短 interval 测试失败
- **首次启动立即刷 1 次** — 用户体验优化,避免"启动后 30s 内 badge 不动"
- **复用 _stop_hotkey_poll Event** — 简化退出(主进程退 → 全停),不引入新 Event
- **`_FakeBadgeConfirmService` 加 `threading.Lock`** — 多线程 polling + 主线程 assert 共享状态必须加锁
