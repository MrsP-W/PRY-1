## 3. 5 段关键设计决策

### 决策 1:`dependencies` 而非 `optional-dependencies.oauth`

- **原因**:沿 launch plan §5.5,msal + google-auth 是 outlook/gmail SMTP **必需**,不是可选
- **代价**:即使不用 outlook/gmail 的用户也会装(纯 Python,0 系统影响)
- **替代方案**:`[oauth]` optional 段 + 显式 `uv pip install .[oauth]` → 拒(增加安装步骤,与 v0.2.1 #6 Phase 1 launch 强契约 `SERVICE_OAUTH_MICROSOFT` / `SERVICE_OAUTH_GOOGLE` 矛盾)

### 决策 2:用 `>=` 锁下限,不锁精确版本

- **原因**:`msal>=1.24` / `google-auth>=2.23` / `google-auth-oauthlib>=1.0` 是**下限**,具体版本由 `uv.lock` 锁
- **代价**:不同用户装可能拿到不同次版本(但 patch 兼容)
- **替代方案**:锁精确版本 `msal==1.37.0` → 拒(失去次版本 bugfix 自动更新)
- **业务价值**:uv.lock 保证 dev/CI/prod 一致(关键 commit 5 增量价值)

### 决策 3:不加 `msal-extensions` / `google-auth-httplib2`

- **原因**:`msal-extensions` 用于 token cache(需 `msal[cache]`),`google-auth-httplib2` 是旧 transport 层
- **代价**:用户实现 token cache 需自己用 `msal[cache]` extra 或 keyring
- **替代方案**:加 `msal[cache]` 直接用 OS token cache → 拒(本项目用 Keychain,跨平台不一致)
- **业务价值**:依赖图最小化,避免引入 httplib2 旧版依赖

### 决策 4:不加 `cryptography` / `pyjwt`(google-auth 传递依赖)

- **原因**:`google-auth` 依赖 `cryptography` + `pyjwt`,uv.lock 自动解
- **代价**:wheel 体积略增(~1MB)
- **替代方案**:显式 pin `cryptography>=41` → 拒(传递依赖由 google-auth 管,不要重复声明)
- **业务价值**:依赖图简洁(google-auth 自己管传递依赖)

### 决策 5:不开 `optional-dependencies.oauth`(避免双重真相源)

- **原因**:开 `[oauth]` 段会让"用户装哪个版本"出现分支,增加测试矩阵(无 oauth / 有 oauth 两种环境)
- **代价**:所有用户都装 msal+google-auth(纯 Python,0 性能损耗)
- **替代方案**:`[oauth]` 段 + 文档注明 outlook/gmail 用户需装 → 拒(commit 5 阶段不开,等 OutlookProvider/GmailProvider 决策)
- **业务价值**:0 文档分支,所有用户一致

## 5. 5 条关键教训沉淀

### 5.1 范本 1:1 复用 launch plan §2.1 commit 5 计划

完全按 launch plan §2.1 commit 5 实施:
- `pyproject.toml` 加 3 依赖
- `uv lock` 锁版本
- 8/8 质量门验证
- 单 commit 收口

**无 scope creep,无意外 B 类决策**。建议:**任何 commit 严格按 launch plan 计划,不夹带**。

### 5.2 `>=` 下限 + `uv.lock` 精确锁 = dev/CI/prod 一致

`pyproject.toml` 写 `>=` 下限(允许次版本 bugfix 自动更新),`uv.lock` 锁精确版本(保证环境一致)。建议:**永远用 `>=` + `uv lock` 组合**,不要锁精确版本。

### 5.3 纯 Python wheel 0 系统影响

msal + google-auth + google-auth-oauthlib 全部**纯 Python wheel**,无 native 依赖,无系统库要求。
- 用户装 `uv pip install .` 自动解
- macOS / Linux / Windows 表现一致
- 不需要 C 编译器

建议:**新增 OAuth 类库,优先选纯 Python wheel**。

### 5.4 `dependencies` 而非 `optional-dependencies.oauth` 的代价评估

- 优点:0 文档分支,所有用户一致
- 代价:即使不用 outlook/gmail 也装 msal + google-auth(~5MB)

**当前决策接受 5MB 代价换 0 文档分支**(commit 5 阶段不开 optional,等 OutlookProvider/GmailProvider 决策再评估)。建议:**B 类决策延后**:是否切换到 `[oauth]` 段 = 独立议题,等用户决定 outlook/gmail 是否启用。

### 5.5 commit 5 是"无 new tests commit"的范本

commit 5 仅加依赖 + uv.lock,**0 new tests**(因为 commit 2/3/4 已 mock 化覆盖)。这正是 v0.2.2 范本的精华:**每个 commit 单一职责,加 dep 单独 commit,加 mock 单独 commit**。
