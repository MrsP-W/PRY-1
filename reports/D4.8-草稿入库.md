**v1.0.1 关键修复 2 项**(沿用 D4.7.3 v1.0.5 P2-2 范本 + task_packet 真实签名核对):

1. **LaneBoard.add 拒绝 FINISHED 终态**(commit `e3f0d80` L107-127):
   - **现象**:`store_and_emit` 第 7 步首次 `LaneBoard.add` 传入 `LaneStatus.FINISHED if business_accepted else BLOCKED`,业务成功时抛 `PolicyLaneError: 新 entry 不能是 FINISHED 状态(终态, 需先走 ACTIVE/BLOCKED 路径)`
   - **根因**:`LaneBoard.add` 是新增入口,只接受 `ACTIVE` / `BLOCKED` 状态(终态 `FINISHED` 必须经 `update` 转换),与 D4.7.4 v1.0.2 范本一致(成功/业务阻断/技术失败 3 入口都先 add ACTIVE → update 终态)
   - **修复**:首次 add 改用 `LaneStatus.ACTIVE`(即使是 FINISHED 终态,先 ACTIVE 入表),然后 `update` 到 `FINISHED` / `BLOCKED`(合法转换 ACTIVE → FINISHED/BLOCKED)。3 入口范本统一。
2. **`recovery_policy` 白名单修正**(commit `e3f0d80` L302/389):
   - **现象**:`build_outbox_blocked_packet` / `build_outbox_failure_packet` 用 `"never_retry"` / 写死字符串,触发 `PolicyContractError: recovery_policy 非法: 'never_retry' 不在 {'none', 'retry_on_transient', 'manual'}`
   - **根因**:`task_packet.TaskPacket` 的 `recovery_policy` 字段是 `Literal['none', 'retry_on_transient', 'manual']` 白名单,D4.7.4 v1.0.2 范本已用 `"none"` / `"retry_on_transient"`,D4.8 v1.0 初版写错白名单
   - **修复**:`build_outbox_blocked_packet` 改 `"none"`(业务阻断永不重试,cf=0 锁死),`build_outbox_failure_packet` 改 `"retry_on_transient"`(技术失败可重试,cf>=1)

## 4. ⚠️ 已知限制(D4.8.1+ 复检 P 项,5 段待人工审批)

### 4.1 限制 1:`OutboxPriority` 仅 3 类,缺 URGENT/HIGH/LOW

- `OutboxPriority` 当前 3 类:`NORMAL` 默认 / `HIGH` / `URGENT`(L420-422)
- 真实业务可能需要 4-5 级(如 CRITICAL > URGENT > HIGH > NORMAL > LOW)
- **B 类决策**,D4.8.1+ 扩 priority 枚举

### 4.2 限制 2:outbox 无 SLA 字段(超时未发送无告警)

- 当前 11 字段无 `sla_due_at` / `timeout_at`
- 实际场景:草稿入库 1 小时后未发送应告警
- **B 类决策**,D4.8.1+ 加 `sla_due_at` 字段 + 告警逻辑

### 4.3 限制 3:真实 SMTP 发送未做(只入库,不真发)

- `store_and_emit` 成功路径只 `insert outbox row`,0 接触 SMTP
- D4.8 锁定不依赖真实发送
- **B 类决策**,D4.9 调度器消费 `status='pending'` 行后真发

### 4.4 限制 4:`blacklist_recipients` 配置表未做(无域名/邮箱阻断)

- 当前 `recipient_email` 0 阻断,可向任意地址入库
- 真实场景:屏蔽 `noreply@*` / 内部测试邮箱
- **B 类决策**,D4.8.1+ 加 `blacklist_recipients` 配置表

### 4.5 限制 5:状态机转换未做显式校验

- `update_status` 当前接受任意 status → status 转换
- 实际场景:`sent` 不能转 `pending`(终态,不可逆)
- **B 类决策**,D4.8.1+ 加状态机约束(合法转换:`pending → sending → sent` / `pending → failed` / `sending → failed`)
