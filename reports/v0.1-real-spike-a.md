## 3. Spike 5 详细(launchd 部署)

### 3.1 install 跑通

| 步 | 命令 | 结果 |
|---|------|------|
| 1 | `bash scripts/launchd_install.sh install` | ✅ 文件部署全部成功(`~/bin/my-ai-employee-monthly-report` + `~/Library/LaunchAgents/com.myaiemployee.agent.plist` + `~/Library/Logs/MyAIEmployee/`),**launchctl load 实际成功**(后续 launchctl list 已显示注册),脚本 5 源验证 L114 race condition 触发 exit 3(误报)|
| 2 | 5 维度验证 deploy 状态 | ✅ `launchctl list` 显示 `- 0 com.myaiemployee.agent`(已注册,未运行 — `RunAtLoad=false` 且非 1 号 09:00)+ plist 2366 bytes + `~/bin/my-ai-employee-monthly-report` 197 bytes 可执行 + 日志目录就绪(2 个 0 字节 log) |

### 3.2 uninstall + 清理

| 步 | 命令 | 结果 |
|---|------|------|
| 3 | `bash scripts/launchd_install.sh uninstall` | ❌ **脚本未实现 uninstall 函数** — 承诺了 install/uninstall 双模式,只实现了 install(脚本 L69-130 全部为 install 逻辑) |
| 4 | 手动 cleanup:`launchctl unload` + `rm -f plist` + `rm -f ~/bin/...` + `rm -rf ~/Library/Logs/MyAIEmployee/` | ✅ launchctl unload exit 0 + plist 删 + `~/bin/` 删 + 日志目录删 + 最终 `launchctl list \| grep myaiemployee` 空(无残留)|

### 3.3 关键发现

1. **`launchctl load -w` 实际成功**(`launchctl list` 显示 `- 0 com.myaiemployee.agent` = "已注册,未运行" 状态,PID `-` 因 `RunAtLoad=false`),脚本 5 源验证 L114 误判"未见"是 launchctl 缓存 race condition(立刻 grep 比 load 同步慢)。
2. **`launchctl print user/$(id -u)/com.myaiemployee.agent` 报 "Could not find service"** — macOS Sequoia/Tahoe launchctl 域命令变化(沿 D5.6 5 源判定范本,launchctl list 是唯一稳定探测)。plist docstring L18 注释"launchctl print 必能读"是 doc drift。
3. **`launchd_install.sh` 缺少 `uninstall` 函数**(P2 finding,留作 D10+ 修复)— 脚本 README/docstring 承诺 `install/uninstall` 双模式,实装只支持 install,导致 spike 收尾必须手动 cleanup。
4. **`launchctl load -w` 重复 load 报 "Load failed: 5: Input/output error"** — 因为 plist 已注册,二次 load 失败但不影响运行状态(macOS launchd 行为)。

### 3.4 教训

- **deploy 状态必须以 `launchctl list` 为唯一真理源**,不依赖 `launchctl print`(macOS 域命令不稳定)
- **PID `-` 不代表"未注册"** — launchd "已注册,未运行" 状态(`RunAtLoad=false` 或非触发时段)
- **脚本承诺双模式必须实装**:`install/uninstall` 不能只写一半,否则 spike 收尾污染用户环境
- **手动 cleanup 4 步范本**(load 卸载 + plist 删 + `~/bin/` 删 + 日志删)是 launchd 部署清理的标准动作,留作 D10+ 文档化
