## 命名规范

- `D{N}-{主题}.md` — Day 级报告（如 `D1-脚手架完成.md`）
- `W{N}-{主题}.md` — Week 级报告（如 `W1-MVP完成.md`）
- `M{N}-{主题}.md` — Milestone 级报告（如 `M1-v0.1发布.md`）
- `S{YYYY-MM}-{主题}.md` — 特别事件（如 `S2026-07-重大事故复盘.md`）

## 引用规则

- 主文档（README/architecture/week1/week2）**不直接放**，仅在相关章节 `📌 下一棒` 引用
- 阶段报告**只读历史**，不再修改；修正放新报告并 `supersedes:` 标注
- 报告中的 D2+ 启动建议**作为参考**，实际启动需用户明确确认
