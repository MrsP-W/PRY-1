## 3. 修复方案(1 commit · +71 -0)

### 3.1 新增 `tests/db/conftest.py`(沿 events/conftest.py 范本)

```python
# 显式 import 触发 SQLAlchemy Base.metadata 注册
# E402 是必须的(因 sys.path.insert 必须在 import 前);F401 是必须的(只 import 不引用)
from my_ai_employee.core.models import Base  # noqa: E402, F401
from my_ai_employee.core.outbox import (  # noqa: E402, F401
    OutboxEntry,  # 触发 outbox 表注册
)
from my_ai_employee.db.blacklist import RecipientBlacklist  # noqa: E402, F401
from my_ai_employee.db.merchant_profile import MerchantProfile  # noqa: E402, F401
from my_ai_employee.db.notes import Note  # noqa: E402, F401
from my_ai_employee.db.transactions import Transaction  # noqa: E402, F401
from my_ai_employee.events import EventStore  # noqa: E402, F401
from my_ai_employee.events.models import Event  # noqa: E402, F401  触发 events 表注册


@pytest.fixture
def engine() -> Iterator:
    eng = create_engine("sqlite:///:memory:")
    Base.metadata.create_all(eng)  # 所有表 + FK 一次性注册成功
    yield eng
    eng.dispose()
```

**为什么是 6 个 ORM**:
- `OutboxEntry` + `Event`:修复 FK 循环(关键)
- `RecipientBlacklist`:test_blacklist 11 个测试需要
- `MerchantProfile` + `Transaction`:test_merchant_profile 5 个测试需要
- `Note`:test_notes 13 + test_notes_fingerprint 7 + test_notes_l2_cross_source 9 + test_notes_sync_status 12 = 41 个测试需要
- `EventStore`:events 模块完整导入(防后续 test 缺引用)

**为什么必须 E402 + F401**:
- E402:必须在 `sys.path.insert` 之后 import,违反"import 必须在顶部"规则
- F401:只 import 不引用(SQLAlchemy 触发注册靠 import 副作用,变量无引用),违反"未使用 import"规则

## 5. 关键设计决策(5 条)

### 5.1 不改 `tests/db/test_*.py`,只新增 conftest

**理由**:
- 各 test 文件 `engine` fixture 自写 create_all 范本已稳定(沿 tests/events/conftest.py 范本)
- 修复点 1 处(conftest.py),影响 11 个 test 文件
- 风险最低:不动测试逻辑,只确保 Base.metadata 注册所有 ORM

### 5.2 显式 import 触发 SQLAlchemy 注册(范本继承)

**沿用范本**:`tests/events/conftest.py:16-20` 已用 `noqa: F401` 显式 import 触发注册。
`tests/db/conftest.py` 套用同一范本,**统一项目内 SQLAlchemy ORM 测试约定**。

### 5.3 `sys.path.insert` 用 E402 noqa

**理由**:
- 必须先 `sys.path.insert` 才能 `from my_ai_employee.core.models import Base`
- ruff E402 规则触发,但 conftest.py 是特殊场景(测试基础设施)
- 用 `# noqa: E402, F401` 显式标记,留 audit trail

### 5.4 import 6 个 ORM 而非 `import *`

**理由**:
- 显式 import = 显式契约 = 后续 reviewer 一眼看清"conftest 注册了哪些 ORM"
- `import *` 容易漏注册(后续加 model 不知道)
- 删除一个 ORM 时 conftest 必须改(强一致)

### 5.5 不动 11 个 test_*.py(零修改测试逻辑)

**理由**:
- 沿 [[d5.5.5-p1-p2-p3-fixes]] 教训"删依赖慎之又慎"
- 修复 1 处基础设施,影响 11 个 test,范围最小化
- 主测试套 2176 passed 必须不破坏(本项目最高优先级)
