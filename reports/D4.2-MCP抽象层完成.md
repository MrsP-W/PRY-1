## 3. 4 类业务异常设计

### 3.1 异常层级

```python
MCPError (基类, catch-all for MCP 业务异常)
├── MCPTimeoutError        # recoverable=True  → 重试
├── MCPConnectionError     # recoverable=True  → 重试
├── MCPProtocolError       # recoverable=False → 不重试
└── MCPResponseError       # recoverable=False → 不重试
```

**关键设计**：
- **recoverable 标志**：`isinstance(exc, (MCPTimeoutError, MCPConnectionError))` → 决定是否重试
- **不混入 `Exception`**：与 D3.3.3 + D4.1 教训一致，避免掩盖编程错误
- **`error_surface()`** 方法：MCPClient 把任何 `MCPError` 转 `McpErrorSurface`(5 字段)

### 3.2 McpErrorSurface 5 字段

```python
@dataclass(frozen=True)
class McpErrorSurface:
    phase: LifecyclePhase   # DISCOVERY/CONNECT/INITIALIZE/CALL/DISCONNECT
    server: str             # server 名
    message: str            # str(exc)
    context: dict[str, Any] # {"exc_type": "MCPConnectionError", ...}
    recoverable: bool       # 决策依据
```

### 3.3 McpDegradedReport 4 段

```python
@dataclass
class McpDegradedReport:
    working: list[str]              # 连接成功的 server
    failed: list[str]               # 连接失败的 server
    available_tools: set[str]       # 健康 server 暴露的工具
    missing_tools: set[str]         # 失败 server 的 expected_tools
    errors: list[McpErrorSurface]   # 详细错误信息
    # 派生属性
    @property
    def is_healthy(self) -> bool: ...   # failed 为空 + missing 为空
    @property
    def is_degraded(self) -> bool: ...  # failed 非空
```

## 4. 关键决策记录

### 4.1 Required flag 决策

| server | required | 失败行为 | 启动能否继续 |
|--------|----------|---------|-------------|
| filesystem | False | 降级 + missing_tools += ["read_file"] | ✅ |
| calendar | True | 抛 `MCPConnectionError("cal down")` + 关闭已连 server | ❌ |

**为什么 calendar 必填**：业务逻辑里"今天没日历"用户能感知，且 Events 表 D4.3 需要 calendar 工具。filesystem 缺失只是降级（可手动传文件）。

### 4.2 重试只针对 recoverable

```python
for attempt in range(self.max_retries + 1):
    try:
        resp = self.transport.send(request)
        self._validate_response(resp, method="tools/call")
        return resp.get("result", {})
    except (MCPTimeoutError, MCPConnectionError) as e:
        # 可恢复: 重试
        last_error = e
        if attempt < self.max_retries:
            time.sleep(self.retry_backoff * (attempt + 1))
            continue
        raise
    except (MCPProtocolError, MCPResponseError):
        # 不可恢复: 不重试, 直接抛
        raise
```

**为什么 protocol/response 不重试**：协议错重试还是错（response 结构本身坏了），响应错可能是业务错（tool 不存在）重试浪费资源。

### 4.3 D3.3.3 教训应用（3 处）

1. **`except MCPError` 收窄**：4 类业务异常透传，`ValueError`/`TypeError` 直接抛
2. **`call_tool()` 工具名不在列表 → `ValueError`**：编程错误，不包装
3. **`_validate_response()` 明确分类**：protocol vs response 错误区分（前者重试无意义，后者可能是 tool 错）

## 7. 关键 regression 测试

### 7.1 `test_keeps_healthy_servers_when_optional_fails`

**对应 claw-code**：`manager_discovery_report_keeps_healthy_servers_when_one_server_fails`

```python
# 1 个可选 server 失败, 1 个必填 server 成功
discovery.DEFAULT_SERVERS = {
    "fs_optional_failing": ServerConfig(
        name="fs_optional_failing", required=False,  # 可选
        transport_factory=_make_failing_factory(MCPConnectionError("fs broken")),
        expected_tools=["read_file"],
    ),
    "cal_working": ServerConfig(
        name="cal_working", required=True,  # 必填（但成功）
        transport_factory=_make_working_factory("cal_working", ["create_event"]),
        expected_tools=["create_event"],
    ),
}
clients, report = discover_servers()
# 关键: 健康的 cal_working 仍然 connected
assert "cal_working" in clients
assert clients["cal_working"].transport.connected is True
# 失败的 fs_optional_failing 在 failed list
assert "fs_optional_failing" in report.failed
# missing_tools 含 fs 的 expected_tools
assert "read_file" in report.missing_tools
```

### 7.2 `test_call_tool_retries_on_timeout`

**recoverable=True → 重试 max_retries+1 次**

```python
t = MockTransport(server_name="fs", tools=["read_file"])
client = MCPClient(server_name="fs", transport=t, max_retries=2, retry_backoff=0.0)
client.connect()
t._send_log.clear()
t.call_timeout = True
with pytest.raises(MCPTimeoutError):
    client.call_tool("read_file")
# 重试 2 次 + 第 3 次最终尝试 = 3 次 send
call_count = sum(1 for r in t._send_log if r.get("method") == "tools/call")
assert call_count == 3
```

### 7.3 `test_call_tool_does_not_retry_on_protocol_error`

**recoverable=False → 不重试，只 1 次**

```python
t.call_protocol_error = True
with pytest.raises(MCPProtocolError):
    client.call_tool("read_file")
call_count = sum(1 for r in t._send_log if r.get("method") == "tools/call")
assert call_count == 1  # 不重试
```
