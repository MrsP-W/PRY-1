## 4. ⚠️ 已知限制(D4.7.4.1+ 复检 P 项,6 段待人工审批)

### 4.1 限制 1:4 类业务阻断白名单硬编码,不支持自定义

- **现状**:`ReviewBlockReason` 4 类(SEINSITIVE_WORD_HIT/TEMPLATE_VIOLATION/TONE_MISMATCH/FACTUAL_CONFLICT)硬编码在 StrEnum,`local_block` 函数用 4 段 if-elif 串行判定
- **风险**:**用户邮件场景可能需要更多类**(如 LEGAL_RISK / PRIVACY_VIOLATION / BRAND_VOICE 等)
- **缓解**:D4.7.4.1+ 改 4 类 → 6+ 类(**B 类决策**,需用户审批)
- **决策**:✅ **接受现状** — 4 类是 week 1 MVP 最小集,D4.7.4 起始固定,扩枚举需 B 类审批

### 4.2 限制 2:`review_batch` 是顺序串行,不支持并发

- **现状**:`review_batch` 顺序串行调 `review`,避免触发熔断 / 雪崩
- **风险**:**100 封草稿顺序审阅 ≈ 50-500s**(每封 500-3000ms)
- **缓解**:D4.7.4.1+ 用 `asyncio.gather` + httpx async 改并发,目标 100 封 < 60s
- **决策**:✅ **接受现状** — D4.7.4 顺序串行是稳健优先(避免熔断/雪崩),并发留 D4.7.4.1+;Week 2 业务层调方一般 < 10 封,延迟可接受

### 4.3 限制 3:敏感词库内置 20 个高风险词,无外部配置

- **现状**:`_DEFAULT_SENSITIVE_WORDS` frozenset 20 词(PII / 安全 / 内部代号 / 商业秘密 / 硬承诺模式)硬编码
- **风险**:**用户邮件场景需要更多词**(如公司特定合规词 / 行业禁用词 / 客户黑名单)
- **缓解**:D4.7.4.1+ 接入 `sensitive_words` 配置表(SQLite + 配置文件)
- **决策**:✅ **接受现状** — 20 词是 D4.7.4 起始最小集;扩词库需 B 类审批

### 4.4 限制 4:本地阻断优先级 vs LLM 审阅的协同

- **现状**:`local_block` 4 类本地规则先判定,命中即阻断;未命中才调 LLM
- **风险**:**LLM 审阅可能与本地阻断冲突**(如本地放过但 LLM 标 `sensitive_word_hit`)
- **缓解**:D4.7.4.1+ 增加"先 LLM 后本地"二次复核模式
- **决策**:✅ **接受现状** — 本地先阻断是性能优化(避免无效 LLM 调用);LLM 复核留 D4.7.4.1+ 评估

### 4.5 限制 5:无 100 封真实 spike

- **现状**:D4.7.4 测试 95 + 118 都在**mock router / in-memory EventStore** 级别,未跑 100 封真实邮件端到端
- **风险**:**LLM 真实审阅延迟 / 阻断率 / 阻断原因分布**未在 100 封真实数据上验证
- **缓解**:D4.1.1 LLM HTTP 已验证(159 测试 + 单元),D4.7.4 仅增加严判解析 + 业务接入层
- **决策**:⏳ **D4.7.4.10 必跑** — 在 100 封真实审阅上跑 `review_and_emit`,验证 (a) 审阅准确率 > 80% (b) 阻断率 < 30% (c) 4 类阻断原因分布均匀 (d) 审阅延迟 < 5s

### 4.6 限制 6:CLI 集成未做

- **现状**:`EmailReviewer.review()` 主入口是 Python API,无 `mmx review <draft_id>` 子命令
- **风险**:**用户无法在终端审阅草稿**,只能写 Python 脚本调 API
- **缓解**:D4.7.4 暴露完整 API,业务层 + 未来 Web UI 可直接调
- **决策**:✅ **接受现状** — D4.7.4.1+ 加 `mmx review` 子命令(同 D4.5.1+ `mmx policy status` 模式)

### 4.7 限制 7(D4.7.3 v1.0.6 已应用,本步零回归):Adapter 主类用 `or` 注入替身被吞

- **D4.7.3 v1.0.3 修复**:`engine or PolicyEngine()` 改 `engine if engine is not None else PolicyEngine()`(`integration.py` D4.7.3 段,所有 3 个 Adapter 同步)
